## Scope

E3.1 has four concrete deliverables:

1. **`ProviderPlugin` trait** — the Rust plugin contract: `id`, `track`, `flush`, `shutdown`. Any crate implementing this trait becomes a valid analytics destination.

2. **`ProviderRegistry`** — holds registered plugins, dispatches `TrackEvent` to all of them in order. Synchronous for E3.1; async batching lands in E3.3.

3. **`ProviderConfig` expansion** — `config.rs` already has a stub `ProviderConfig { name: String }`. E3.1 firms it up: adds `config: HashMap<String, String>` for key-value provider options (API keys, endpoints).

4. **Codegen update** — the generated TypeScript `trackXxx()` functions currently throw a stub error. E3.1 replaces the stub with a real runtime:
   - `Provider` TypeScript interface
   - `InfergenConfig` TypeScript interface
   - Module-level `_providers: Provider[]`
   - `configureInfergen(config: InfergenConfig): void` export
   - `trackXxx()` dispatches via `_providers.forEach(p => p.track(...))`

> E3.2 (first-party adapters) implements Segment/PostHog/etc. E3.3 (delivery engine) adds async batching + retry. E3.1 defines only the contract and wires up the TypeScript runtime stub.

## File Specs

### 1. `crates/infergen-core/src/provider/mod.rs`

**Purpose:** Define the `ProviderPlugin` trait, `TrackEvent`, and `ProviderRegistry`.

**Exported symbols:**

```rust
use std::collections::HashMap;
use serde_json::Value;

/// A single analytics event ready to dispatch to providers.
pub struct TrackEvent {
    /// Event name, e.g. `"page_viewed"`.
    pub name: String,
    /// Typed properties keyed by name.
    pub properties: HashMap<String, Value>,
}

/// Contract that every analytics provider adapter must satisfy.
///
/// Implement this trait to create a custom destination. See E3.2 for
/// first-party implementations (Segment, PostHog, Amplitude, …).
pub trait ProviderPlugin: Send + Sync {
    /// Unique, lowercase, hyphen-separated provider ID, e.g. `"posthog"`.
    fn id(&self) -> &str;

    /// Send a single event to this provider.
    ///
    /// # Errors
    /// Returns `Err` if the provider cannot accept the event (e.g. not
    /// initialised, serialization failure). Callers (the registry) will
    /// propagate the first error encountered.
    fn track(&self, event: &TrackEvent) -> crate::Result<()>;

    /// Flush any internally buffered events.
    ///
    /// Default: no-op. Override in providers that buffer.
    fn flush(&self) -> crate::Result<()> {
        Ok(())
    }

    /// Gracefully shut down the provider (flush + close connections).
    ///
    /// Default: no-op. Override in providers with open resources.
    fn shutdown(&self) -> crate::Result<()> {
        Ok(())
    }
}

/// Holds registered [`ProviderPlugin`]s and fans out tracking calls.
pub struct ProviderRegistry {
    plugins: Vec<Box<dyn ProviderPlugin>>,
}

impl ProviderRegistry {
    /// Create an empty registry.
    pub fn new() -> Self { ... }

    /// Register a provider. Later registrations are dispatched after earlier ones.
    pub fn register(&mut self, plugin: Box<dyn ProviderPlugin>) { ... }

    /// Send `event` to every registered provider in registration order.
    ///
    /// Returns the first error encountered; subsequent providers are still called.
    /// Returns `Ok(())` if all succeed or the registry is empty.
    pub fn track(&self, event: TrackEvent) -> crate::Result<()> { ... }

    /// Flush all registered providers. Returns the first error, if any.
    pub fn flush(&self) -> crate::Result<()> { ... }

    /// Shut down all registered providers. Returns the first error, if any.
    pub fn shutdown(&self) -> crate::Result<()> { ... }

    /// `true` if no providers are registered.
    pub fn is_empty(&self) -> bool { ... }

    /// Number of registered providers.
    pub fn len(&self) -> usize { ... }

    /// IDs of all registered providers, in registration order.
    pub fn provider_ids(&self) -> Vec<&str> { ... }
}

impl Default for ProviderRegistry { ... }
```

**Key logic:**

- `track`: iterates `plugins`, calls `.track(&event)` on each, captures first error, continues to remaining plugins, returns first error at end (or `Ok(())`).
- `flush` / `shutdown`: same pattern — call all, return first error.
- `ProviderRegistry::default()` = `ProviderRegistry::new()`.

**Errors:** Uses `crate::Result<T>` (= `Result<T, crate::Error>`). Provider failures are `Error::Io` (network I/O) or a new variant added in `lib.rs`.

**New error variant in `lib.rs`:**
```rust
/// A provider plugin returned an error during tracking.
#[error("provider {id:?} failed: {message}")]
ProviderError {
    id: String,
    message: String,
}
```

**Dependencies:** `std::collections::HashMap`, `serde_json::Value` (already in `infergen-core` deps), `crate::{Result, Error}`.

**Unit tests (in `mod tests`):**

| Test | What it proves |
|------|---------------|
| `registry_new_is_empty` | `new()` → `is_empty() == true`, `len() == 0` |
| `registry_register_increments_len` | register 2 plugins → `len() == 2` |
| `registry_provider_ids_in_order` | register "a" then "b" → `provider_ids() == ["a", "b"]` |
| `registry_track_calls_all_plugins` | `track` dispatches to all; both plugins record the event |
| `registry_track_returns_first_error` | first plugin fails; second still called; first error returned |
| `registry_flush_calls_all` | `flush` calls all plugins' `flush()` |
| `registry_shutdown_calls_all` | `shutdown` calls all plugins' `shutdown()` |
| `registry_empty_track_ok` | track on empty registry → `Ok(())` |
| `default_trait_impls` | default `flush()` and `shutdown()` return `Ok(())` |

### 4. `crates/infergen-core/src/codegen/mod.rs`

**Purpose:** Replace the `throw new Error(...)` stub with a full TypeScript runtime preamble: `Provider` interface, `InfergenConfig`, `_providers` variable, and `configureInfergen` function. Update `trackXxx()` body to dispatch via `_providers.forEach`.

**New TypeScript output structure (all events):**

```
// Auto-generated header (unchanged)

// NEW: runtime preamble block
/** A provider that receives analytics events. */
export interface Provider {
  /** Unique provider identifier, e.g. "posthog". */
  id: string;
  /** Send a single analytics event. */
  track(eventName: string, properties: Record<string, unknown>): void;
  /** Flush buffered events (optional). */
  flush?(): void;
  /** Shut down the provider (optional). */
  shutdown?(): void;
}

/** Runtime configuration. Pass to `configureInfergen` at app startup. */
export interface InfergenConfig {
  /** Analytics providers that receive every `track` call. */
  providers: Provider[];
}

/** @internal Registered providers — populated by `configureInfergen`. */
let _providers: Provider[] = [];

/** Configure the Infergen runtime. Call once before any track calls. */
export function configureInfergen(config: InfergenConfig): void {
  _providers = config.providers;
}

// EventName union (unchanged)
// Per-event sections (interface unchanged; function body changes)

// OLD function body:
//   throw new Error("Infergen runtime not configured...");
// NEW function body:
//   _providers.forEach(p => p.track("event_name", properties));
```

**Implementation changes in `codegen/mod.rs`:**

1. Extract `write_runtime_preamble(out: &mut String)` — writes the four blocks above (Provider interface, InfergenConfig, `_providers`, `configureInfergen`).
2. Call `write_runtime_preamble(&mut out)` from `generate_typescript`, after the header and before `build_event_name_type`.
3. Change `write_function` to emit `_providers.forEach(p => p.track("{name}", properties));` instead of the `throw new Error(...)` block.

## Open Questions / Risks

- **`track` error semantics:** Returns first error but continues dispatching to remaining providers. This is intentional for E3.1 — don't silently swallow errors, but don't abort other providers either. E3.3 adds proper per-provider error recovery.
- **`_providers` is module-level mutable state:** This is idiomatic for a "configure once, use everywhere" SDK pattern. The TypeScript `let` (not `const`) is necessary. ESLint `no-var` rules are fine with `let`.
- **`ProviderConfig.config` TOML serialization:** `HashMap<String, String>` serializes as an inline table in TOML. With `skip_serializing_if = "is_empty"`, empty maps are omitted — round-trip safe.
- **Breaking the `ProviderConfig` struct literal in `config.rs` tests:** Fixed by adding `#[derive(Default)]` + `..Default::default()` in the test.
- **`generate_empty_catalog_has_event_name_never` (existing test):** Still passes — the preamble is prepended before `EventName = never`. No assertion conflict.
- **`generate_check_up_to_date_exits_zero` (E2.3 CLI test):** Still passes — both generate calls produce the new format, so they match.
- **Clippy `collapsible_if` in `provider/mod.rs`:** Three nested `if let Err(e) ... { if is_none() { ... } }` patterns collapsed to `if let Err(e) ... && first_err.is_none() { ... }` using let-chains (stable since Rust 1.88, workspace requires 1.94).
