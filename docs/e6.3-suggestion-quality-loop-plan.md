---

## What This Epic Delivers

A local feedback loop that learns from every `infergen review approve` / `ignore` decision:

1. **Confidence tuning** — approvals and ignores are aggregated per `(adapter, kind)`. Future proposals from consistently-ignored signals get lower confidence; from consistently-approved signals get a small boost.
2. **Name hints** — when the user approves an event on a particular source file, the approved name is recorded. On the next scan, if the same source path + kind reappears, the historical name is pre-populated instead of re-running the heuristic, reducing naming churn.
3. **Persistence** — stored in `.infergen/quality.yaml` (alongside `catalog.yaml`). Can be committed to share learning across a team, or gitignored to keep it local.
4. **Fully local / offline** — no external calls, no telemetry.

---

## Open Questions / Risks

- **Source path stability**: name hints are keyed on exact relative path. If a file is moved, the hint is silently lost (acceptable for v1).
- **Sparse data**: minimum sample size = 3 entries before applying a confidence multiplier. Below that, return 1.0 (no adjustment). Avoids volatile calibration from one or two decisions.
- **Feedback save failure**: never break the review command. All feedback writes are soft failures (log to stderr, continue).
- **Bulk approve (`--all`)**: records feedback for every approved entry. Could write many entries at once; bounded by catalog size.

### 1. `crates/infergen-core/src/quality/mod.rs`

**Purpose:** Feedback store — persists review decisions, derives confidence multipliers and name hints for future scans.

**Exported symbols:**

```rust
/// Schema version for quality.yaml.
pub const QUALITY_SCHEMA_VERSION: u32 = 1;

/// A single review decision recorded at approve/ignore time.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct FeedbackEntry {
    /// Stable catalog ID of the reviewed event.
    pub event_id: String,
    /// Event name at review time (may differ from heuristic name if renamed).
    pub event_name: String,
    /// Review action taken.
    pub action: FeedbackAction,
    /// Adapter that proposed the event, e.g. `"nextjs"`.
    pub adapter: String,
    /// Event kind in camelCase, e.g. `"pageView"`.
    pub kind: String,
    /// Source path relative to project root.
    pub source_path: String,
    /// Adapter confidence at review time (`0.0`–`1.0`).
    pub confidence_at_review: f64,
}

/// Whether the event was approved or ignored.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub enum FeedbackAction {
    Approved,
    Ignored,
}

/// Feedback store — the `.infergen/quality.yaml` document.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize, Default)]
#[serde(rename_all = "camelCase")]
pub struct FeedbackStore {
    #[serde(default = "default_quality_version")]
    pub schema_version: u32,
    #[serde(default)]
    pub entries: Vec<FeedbackEntry>,
}

impl FeedbackStore {
    pub fn load(path: &Path) -> Result<Self, Error>
    pub fn save(&self, path: &Path) -> Result<(), Error>
    pub fn record(&mut self, entry: FeedbackEntry)

    /// Confidence multiplier for proposals from `(adapter, kind)`.
    ///
    /// Returns 1.0 if fewer than MIN_SAMPLE entries exist for this key.
    /// Otherwise: multiplier = 0.75 + 0.35 * approval_rate.
    ///   - 100% approved → 1.10 (boost; caller clamps to 1.0)
    ///   - 50% approved  → 0.925
    ///   -  0% approved  → 0.75 (25% reduction)
    pub fn confidence_multiplier(&self, adapter: &str, kind: &str) -> f64

    /// Most-recently approved event name for `(adapter, kind, source_path)`.
    ///
    /// Returns `None` if no matching approved entry exists.
    /// Only matches `FeedbackAction::Approved` entries.
    pub fn name_hint(&self, adapter: &str, kind: &str, source_path: &str) -> Option<String>
}

/// Derive quality.yaml path from the catalog path (same directory, different name).
pub fn quality_path(catalog_path: &Path) -> PathBuf
```

**Key logic:**

`load`: if file doesn't exist → `Ok(FeedbackStore::default())`. Parse YAML; propagate `Error::CatalogParse` on malformed content.

`save`: create parent dirs if needed. Serialize to YAML. Propagate `Error::Io` on write failure.

`record`: push entry to `self.entries`. No deduplication — multiple reviews of the same event are all kept (history).

`confidence_multiplier` pseudocode:
```rust
const MIN_SAMPLE: usize = 3;
let relevant: Vec<_> = entries where adapter == adapter && kind == kind
if relevant.len() < MIN_SAMPLE { return 1.0 }
let n_approved = relevant.iter().filter(|e| e.action == Approved).count();
let rate = n_approved as f64 / relevant.len() as f64;
(0.75 + 0.35 * rate).clamp(0.5, 1.1)
// Note: clamp upper bound 1.1 — caller clamps proposal.confidence to 1.0
```

`name_hint` pseudocode:
```rust
entries
    .iter()
    .rev()                         // most recent first
    .filter(|e| e.action == Approved
         && e.adapter == adapter
         && e.kind == kind
         && e.source_path == source_path)
    .map(|e| e.event_name.clone())
    .next()
```

`quality_path`: replace filename component with `"quality.yaml"`:
```rust
catalog_path.with_file_name("quality.yaml")
```

**Edge cases:**
- `quality.yaml` absent: `load` returns empty store, not an error.
- `entries` empty: `confidence_multiplier` returns 1.0.
- Multiple approved entries for same source_path: `.rev()` gives most recent.
- `adapter` field empty string (old proposals): treated as separate key from "nextjs" etc.

**Dependencies:** `serde`, `serde_yaml`, `infergen-core::Error`

### 3. `crates/infergen-cli/src/commands/review.rs` (modify)

**Purpose:** Record a `FeedbackEntry` into `quality.yaml` whenever an approve or ignore action succeeds.

**Changes:**

Add a private helper:

```rust
fn feedback_entries_for_approve(catalog: &Catalog, id: Option<&str>, all: bool)
    -> Vec<FeedbackEntry>
```

Collects `FeedbackEntry` values from the catalog before the mutation:
- `all = true`: collect from all `status == Proposed` entries.
- `all = false, id = Some(id)`: collect from the single matching entry.
- Uses `entry.provenance.first()` for `source_path` and `adapter`.
- Sets `action = FeedbackAction::Approved`.

Add a similar helper for ignore:

```rust
fn feedback_entry_for_ignore(catalog: &Catalog, id: &str) -> Option<FeedbackEntry>
```

Add a shared save helper:

```rust
fn save_feedback(catalog_path: &Path, entries: &[FeedbackEntry])
```

Loads `FeedbackStore` (or creates default), records each entry, saves. Silently ignores all errors with `eprintln!` — never propagates.

**Modified match arms:**

`ReviewAction::Approve { id, all }`:
```rust
// 1. Load catalog to read entry metadata before mutation.
let catalog_for_feedback = load_catalog_soft(&args.catalog);
let feedback = catalog_for_feedback
    .map(|c| feedback_entries_for_approve(&c, id.as_deref(), all))
    .unwrap_or_default();

// 2. Apply mutation (unchanged).
mutate_catalog(&args.catalog, |cat| { ... })?;

// 3. Save feedback (soft failure).
if !feedback.is_empty() {
    save_feedback(&args.catalog, &feedback);
}
```

`ReviewAction::Ignore { id }`:
```rust
let catalog_for_feedback = load_catalog_soft(&args.catalog);
let feedback = catalog_for_feedback
    .as_ref()
    .and_then(|c| feedback_entry_for_ignore(c, &id));

mutate_catalog(&args.catalog, |cat| { ... })?;

if let Some(entry) = feedback {
    save_feedback(&args.catalog, &[entry]);
}
```

`load_catalog_soft` is a private helper that returns `Option<Catalog>` — `None` on any error (avoids breaking review for feedback writes):
```rust
fn load_catalog_soft(path: &Path) -> Option<Catalog> {
    load_catalog(path).ok()
}
```

**Edge cases:**
- Catalog missing → `load_catalog_soft` returns `None`, no feedback recorded, mutation proceeds normally.
- Quality store save fails → `eprintln!` to stderr, mutation result unaffected.
- `approve --all` with 0 proposed events → empty feedback vec, `save_feedback` is not called.
- Entry has no provenance → `source_path = ""`, `adapter = ""` (still stored; `name_hint`/`confidence_multiplier` lookup just won't match future proposals with non-empty paths).

**Dependencies:** `infergen_core::{FeedbackAction, FeedbackEntry, FeedbackStore, quality_path}`, `infergen_core::catalog::load_catalog`

### 4. `crates/infergen-cli/src/commands/scan.rs` (modify)

**Purpose:** Load the feedback store before building proposals and apply two adjustments: confidence multipliers and name hints.

**Changes — add immediately after `all_proposals` is fully built, before catalog merge/creation:**

```rust
// Load feedback store (soft failure — missing store is normal for new projects).
let quality_path = infergen_core::quality_path(&catalog_path);
let feedback = infergen_core::FeedbackStore::load(&quality_path).unwrap_or_default();

// Apply confidence multipliers from historical approval/ignore rates.
for proposal in &mut all_proposals {
    if !proposal.adapter.is_empty() {
        let kind = kind_str(proposal.kind); // "pageView", "apiCall", …
        let m = feedback.confidence_multiplier(&proposal.adapter, &kind);
        proposal.confidence = (proposal.confidence * m as f32).clamp(0.0, 1.0);
    }
}

// Apply name hints from previously approved events on the same source file.
for proposal in &mut all_proposals {
    if !proposal.adapter.is_empty() {
        let rel = proposal
            .source_path
            .strip_prefix(&cwd)
            .unwrap_or(&proposal.source_path)
            .to_string_lossy();
        let kind = kind_str(proposal.kind);
        if let Some(name) = feedback.name_hint(&proposal.adapter, &kind, &rel) {
            proposal.name = name;
        }
    }
}
```

Add a private helper to translate `EventKind` to camelCase string (same mapping as in `refine.rs`):

```rust
fn kind_str(kind: EventKind) -> String {
    match kind {
        EventKind::PageView    => "pageView",
        EventKind::ApiCall     => "apiCall",
        EventKind::AuthEvent   => "authEvent",
        EventKind::FormSubmit  => "formSubmit",
        EventKind::ButtonClick => "buttonClick",
        EventKind::Search      => "search",
        EventKind::Error       => "error",
    }.to_owned()
}
```

Note: `infergen_core::EventKind` (adapter kind) is already re-exported from `infergen-core`. Import it at top of scan.rs.

Also add import at top:
```rust
use infergen_core::{EventKind, FeedbackStore, quality_path};
```

**Edge cases:**
- `quality.yaml` absent → `FeedbackStore::load` returns default (empty store), adjustments are no-ops.
- `proposal.adapter` empty → skip both adjustments for that proposal.
- Multiplier = 1.0 (no data) → `confidence * 1.0 = confidence`, no change.
- Name hint exists → overrides heuristic name. On next rescan, if the same source file still appears, hint is re-applied deterministically.
- `proposal.source_path` not under `cwd` (edge case with symlinks/absolute paths) → `strip_prefix` fails → `to_string_lossy()` of full path used, which won't match relative-path hints (hint silently skipped — correct behavior).

**Dependencies:** `infergen_core::{EventKind, FeedbackStore, quality_path}`
