## Open Questions / Risks

- **Provenance update on match**: When an existing ID matches a new proposal, we keep the existing entry completely (including old provenance). If a file moves, its ID changes (stable ID includes path), so this is safe. Provenance refresh on matched IDs is deferred to a later epic.
- **Framework detection in scan**: E4.1 uses `NextjsAdapter` for all `.ts/.tsx` files. Generic adapters for other frameworks land in E5.x. Scan emits a notice if no config exists and falls back to defaults.
- **File discovery without walkdir**: Uses `std::fs::read_dir` recursive walk; no new dependencies. Skips `node_modules`, `.git`, `dist`, `target`, `.next`.
- **Scan command flag**: `Scan` is a unit variant in `cli.rs` (no args). E4.1 keeps it as-is; `--dir` and `--catalog` overrides are E4.2 scope.

**Purpose:** Add `rescan_merge` — the three-way merge function keyed on stable IDs.

### Detailed Spec

#### `rescan_merge` signature

```rust
/// Merge fresh scan proposals into an existing catalog using three-way merge semantics.
///
/// ## Merge rules (keyed on stable ID)
///
/// | Scenario | Action |
/// |---|---|
/// | ID in proposals only (new detection) | Add as `Proposed` |
/// | ID in both existing and proposed | Keep existing entry verbatim (all human edits preserved) |
/// | ID in existing only, status `Proposed` | Remove — unreviewed event no longer detected |
/// | ID in existing only, status `Approved` or `Ignored` | Keep — human reviewed |
///
/// Returns a new [`Catalog`] sorted by ID for stable diffs.
#[must_use]
pub fn rescan_merge(
    existing: &Catalog,
    proposals: &[ProposedEvent],
    project_root: &Path,
) -> Catalog
```

#### Algorithm

```
1. Build proposed catalog: from_proposals(proposals, project_root)
   — Reuses existing private conversion machinery for stable-ID generation.

2. Build lookup maps:
   existing_ids: HashSet<&str> from existing.events
   proposed_ids: HashSet<&str> from proposed_catalog.events

3. merged: Vec<CatalogEntry> = Vec::new()

4. For each entry in existing.events:
   - If entry.id in proposed_ids → push existing entry (preserve all human edits)
   - Else (disappeared from scan):
     - EventStatus::Proposed  → skip (drop it)
     - EventStatus::Approved | EventStatus::Ignored → push existing entry (keep)

5. For each entry in proposed_catalog.events:
   - If entry.id NOT in existing_ids → push proposed entry (new detection, status=Proposed)

6. merged.sort_by(|a, b| a.id.cmp(&b.id))

7. Return Catalog { schema_version: CATALOG_SCHEMA_VERSION, events: merged }
```

#### Key Logic / Edge Cases

- **Matched entry** — entire existing `CatalogEntry` is kept verbatim, including `name`, `description`, `status`, `properties`, `providers`, `provenance`. The scanner's refreshed metadata is intentionally discarded for matched IDs to guarantee no human edits are overwritten.
- **Empty proposals** — all `Proposed` entries in existing are removed; all `Approved`/`Ignored` entries survive. A totally fresh project gets an empty result.
- **Empty existing** — equivalent to `from_proposals`; all proposals added as `Proposed`.
- **Deduplication**: `from_proposals` already deduplicates by stable ID, so the proposal set is clean before the merge.
- **Sort**: output always sorted by ID for stable YAML diffs.

### Dependencies

- Existing private functions: `from_proposals` (reused to build proposed catalog internally), `proposal_to_entry` (already used by `from_proposals`)
- `std::collections::HashSet`

**Purpose:** Implement `infergen scan` — discovers source files, runs the parser+adapter, merges proposals into the catalog using `rescan_merge`.

### Detailed Spec

```rust
pub fn run() -> anyhow::Result<()>
```

#### Full algorithm

```
1. cwd = std::env::current_dir()?

2. config = Config::load_from_dir(&cwd)
          .unwrap_or_else(|_| Config::default())
   — Tolerates missing config; uses defaults (catalog = .infergen/catalog.yaml).

3. source_files: Vec<PathBuf> = collect_source_files(&cwd)
   — See `collect_source_files` below.

4. println!("scan: {} files", source_files.len())

5. parser = JsParser
   all_proposals: Vec<ProposedEvent> = vec![]

   for file_path in &source_files:
     source = fs::read_to_string(&file_path) — skip on error
     parsed = parser.parse(&file_path, &source) — skip on error
     adapter = NextjsAdapter::new(&cwd)
     all_proposals.extend(adapter.analyze(&parsed))

6. catalog_path = cwd.join(&config.catalog)

7. match load_catalog(&catalog_path):
     Ok(existing) →
       merged = rescan_merge(&existing, &all_proposals, &cwd)
       new_count = merged.events IDs not in existing
       removed_count = existing Proposed IDs not in merged
       matched_count = merged.events.len() - new_count
       println!("  {new_count} new events added")
       println!("  {removed_count} events removed (Proposed, no longer detected)")
       println!("  {matched_count} events matched (edits preserved)")
       save_catalog(&merged, &catalog_path)?

     Err(infergen_core::Error::Io(_)) →
       cat = from_proposals(&all_proposals, &cwd)
       println!("  {} events proposed", cat.events.len())
       save_catalog(&cat, &catalog_path)?

     Err(e) → return Err(e.into())

8. println!("catalog saved to {}", catalog_path.display())
9. Ok(())
```

#### `collect_source_files` private helper

```rust
fn collect_source_files(root: &Path) -> Vec<PathBuf>
```

Recursively walks `root` using `std::fs::read_dir`. Skips directories named `node_modules`, `.git`, `dist`, `target`, `.next`. Collects files with extensions: `ts`, `tsx`, `js`, `jsx`. Results are sorted for determinism.

```rust
fn collect_files_recursive(dir: &Path, acc: &mut Vec<PathBuf>) {
    let Ok(entries) = std::fs::read_dir(dir) else { return };
    for entry in entries.flatten() {
        let path = entry.path();
        let name = entry.file_name();
        let name_str = name.to_string_lossy();
        if path.is_dir() {
            if !matches!(
                name_str.as_ref(),
                "node_modules" | ".git" | "dist" | "target" | ".next"
            ) {
                collect_files_recursive(&path, acc);
            }
        } else if let Some(ext) = path.extension() {
            if matches!(ext.to_str(), Some("ts" | "tsx" | "js" | "jsx")) {
                acc.push(path);
            }
        }
    }
}

fn collect_source_files(root: &Path) -> Vec<PathBuf> {
    let mut files = Vec::new();
    collect_files_recursive(root, &mut files);
    files.sort();
    files
}
```

/// rescan preserves approved events (three-way merge: Approved + match → kept)
#[test]
fn scan_rescan_preserves_approved_event() {
    let dir = tempdir().unwrap();
    let pages_dir = dir.path().join("pages");
    std::fs::create_dir_all(&pages_dir).unwrap();
    std::fs::write(
        pages_dir.join("index.tsx"),
        "export default function HomePage() { return null; }",
    ).unwrap();
    // First scan: creates catalog with proposed event
    infergen().current_dir(dir.path()).arg("scan").assert().success();

    // Approve the event
    let catalog_path = dir.path().join(".infergen/catalog.yaml");
    let catalog_yaml = std::fs::read_to_string(&catalog_path).unwrap();
    // Find the event ID (starts with evt_)
    let id_start = catalog_yaml.find("id: \"evt_").unwrap();
    let id_slice = &catalog_yaml[id_start + 5..]; // skip 'id: "'
    let id_end = id_slice.find('"').unwrap();
    let event_id = id_slice[..id_end].to_string();

    infergen()
        .current_dir(dir.path())
        .args(["review", "approve", &event_id])
        .assert()
        .success();

    // Second scan: event still present and still approved
    infergen().current_dir(dir.path()).arg("scan").assert().success();
    let after = std::fs::read_to_string(&catalog_path).unwrap();
    assert!(after.contains("approved"), "approved event should survive rescan");
    assert!(after.contains(&event_id), "approved event id should survive rescan");
}
