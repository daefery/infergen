| Contract | Trait | File |

1. A `plugin` sub-module in `infergen-core` with scaffold template generators.
2. `infergen plugin scaffold <kind> <name>` CLI command — emits a ready-to-compile Rust skeleton.
3. `docs/plugin-sdk.md` — the official Plugin SDK guide.
4. Unit tests for scaffold output and CLI wiring.

**Purpose:** Scaffold template generators for all three plugin types.

#### Detailed spec

**`to_pascal_case(s: &str) -> String`**
- Splits on `-` and `_`, capitalises first char of each segment, concatenates.
- `"debug-log"` → `"DebugLog"`, `"my_adapter"` → `"MyAdapter"`, `"posthog"` → `"Posthog"`.
- Empty string → `"Plugin"`.

**`provider_scaffold(name: &str) -> String`**

Returns a complete, ready-to-compile Rust source string for a minimal `ProviderPlugin` implementation. The string:

```
use infergen_core::{ProviderPlugin, TrackEvent, Result};

pub struct {PascalName}Provider;

impl ProviderPlugin for {PascalName}Provider {
    fn id(&self) -> &str {
        "{name}"
    }

    fn track(&self, event: &TrackEvent) -> Result<()> {
        // TODO: send event.name and event.properties to your destination.
        let _ = event;
        Ok(())
    }
}
```

Where `{PascalName}` = `to_pascal_case(name)` and `{name}` = `name` verbatim.

**`adapter_scaffold(name: &str, framework: &str) -> String`**

Returns a minimal `Adapter` skeleton:

```

use infergen_core::{Adapter, Framework, ParsedFile, ProposedEvent};

pub struct {PascalName}Adapter;

impl Adapter for {PascalName}Adapter {
    fn framework(&self) -> Framework {
        // TODO: add {PascalFramework} to the Framework enum in infergen-core,
        // or use an existing variant if the framework is already listed.
        todo!("map to a Framework variant")
    }

    fn analyze(&self, file: &ParsedFile) -> Vec<ProposedEvent> {
        let _ = file;
        // TODO: inspect file.path and use file.with_js_program / with_py_ast /
        // with_go_source / with_ruby_stmts to detect trackable moments.
        vec![]
    }
}
```

Where `{PascalName}` = `to_pascal_case(name)`, `{PascalFramework}` = `to_pascal_case(framework)`.

**`parser_scaffold(name: &str, language: &str) -> String`**

Returns a minimal `LanguageParser` skeleton:

```
use std::path::Path;
use infergen_core::{Diagnostic, Language, LanguageParser, ParsedFile, Result};

pub struct {PascalName}Parser;

impl LanguageParser for {PascalName}Parser {
    fn parse(&self, path: &Path, source: &str) -> Result<ParsedFile> {
        // TODO: parse `source` using your language's parser library.
        // Non-fatal errors should become Diagnostic entries, not Err(...).
        let diagnostics: Vec<Diagnostic> = vec![];
        Ok(ParsedFile::new_opaque(path, Language::{PascalLanguage}, source.to_owned(), diagnostics))
    }
}
```

Where `{PascalName}` = `to_pascal_case(name)`, `{PascalLanguage}` = `to_pascal_case(language)`.

Note: `ParsedFile::new_opaque` is the constructor that stores source without a typed AST. If a parsed AST variant is needed, the community author adds `with_*` methods analogous to the existing ones.

#### Edge cases
- Any whitespace-only `name` → use `"plugin"` as fallback.
- `name` longer than 64 chars → truncate (scaffold code still valid, just verbose).

#### Dependencies
- `infergen_core` (self — `ProviderPlugin`, `TrackEvent`, `Adapter`, `Framework`, `ParsedFile`, `LanguageParser`, `Diagnostic`, `Language`, `Result`)
- No external crates.

```rust
/// Arguments for `infergen plugin`.
#[derive(Debug, Args)]
pub struct PluginArgs {
    #[command(subcommand)]
    pub action: PluginAction,
}

/// `infergen plugin` sub-commands.
#[derive(Debug, Subcommand)]
pub enum PluginAction {
    /// Scaffold a new plugin skeleton and print it to stdout (or --output file).
    Scaffold {
        /// Plugin type to scaffold: provider, adapter, or parser.
        #[arg(value_enum)]
        kind: PluginKind,
        /// Kebab-case name for the plugin (e.g. `my-provider`).
        name: String,
        /// Optional framework name (required for adapter scaffolds).
        #[arg(long)]
        framework: Option<String>,
        /// Write output to this path instead of stdout.
        #[arg(long, short)]
        output: Option<PathBuf>,
    },
    /// List available plugin types and their trait contracts.
    ListTypes,
}

/// Plugin type selector for `infergen plugin scaffold`.
#[derive(Debug, Clone, Copy, PartialEq, Eq, ValueEnum)]
pub enum PluginKind {
    /// An analytics destination (implements `ProviderPlugin`).
    Provider,
    /// A framework adapter (implements `Adapter`).
    Adapter,
    /// A language parser (implements `LanguageParser`).
    Parser,
}
```

Add `Plugin(PluginArgs)` to `Commands`:
```rust
/// Scaffold and describe plugin extension points.
Plugin(PluginArgs),
```

#### Edge cases
- `scaffold adapter` without `--framework` → handler prints an error message and returns `Err(...)`. The CLI definition allows `framework: Option<String>`; validation is in the handler, not in clap, because clap can't express "required only for this variant" cleanly.

### 5. `crates/infergen-cli/src/commands/plugin.rs`

**Purpose:** Handle `infergen plugin scaffold` and `infergen plugin list-types`.  
**Score: 97%**

#### Exported symbol

```rust
pub fn run(args: PluginArgs) -> anyhow::Result<()>
```

#### Detailed spec

```rust
pub fn run(args: PluginArgs) -> anyhow::Result<()> {
    match args.action {
        PluginAction::Scaffold { kind, name, framework, output } => {
            // 1. Validate name: non-empty, only [a-z0-9_-]
            //    If invalid → anyhow::bail!("Plugin name must be lowercase letters, digits, hyphens, or underscores.")
            // 2. Generate scaffold string
            let source = match kind {
                PluginKind::Provider => provider_scaffold(&name),
                PluginKind::Adapter => {
                    let fw = framework.ok_or_else(|| anyhow::anyhow!(
                        "`infergen plugin scaffold adapter` requires --framework <name>"
                    ))?;
                    adapter_scaffold(&name, &fw)
                }
                PluginKind::Parser => {
                    let lang = name.clone(); // language name = plugin name by convention; user can edit
                    parser_scaffold(&name, &lang)
                }
            };
            // 3. Write to output or stdout
            match output {
                Some(path) => std::fs::write(&path, &source)
                    .with_context(|| format!("could not write to {}", path.display()))?,
                None => print!("{source}"),
            }
            Ok(())
        }
        PluginAction::ListTypes => {
            println!("Available plugin types:");
            println!();
            println!("  provider  Implement `ProviderPlugin` to add an analytics destination.");
            println!("            Trait: infergen_core::ProviderPlugin");
            println!("            Methods: id(), track(), flush() [optional], shutdown() [optional]");
            println!();
            println!("  adapter   Implement `Adapter` to detect trackable moments in a framework.");
            println!("            Trait: infergen_core::Adapter");
            println!("            Methods: framework(), analyze()");
            println!();
            println!("  parser    Implement `LanguageParser` to support a new source language.");
            println!("            Trait: infergen_core::LanguageParser");
            println!("            Methods: parse()");
            println!();
            println!("Run `infergen plugin scaffold <type> <name>` to generate a skeleton.");
            Ok(())
        }
    }
}
```

#### Name validation
Regex-free: iterate chars, reject any char outside `[a-z0-9_-]`. Also reject empty string.

#### Dependencies
- `anyhow` (already in CLI `Cargo.toml`)
- `infergen_core::plugin::{provider_scaffold, adapter_scaffold, parser_scaffold}`
- `std::fs::write`

- **Open questions / risks:**
  - `ParsedFile::new_opaque` does not yet exist — parser scaffold calls it, and the implementation must add this constructor (or equivalent) to `ParsedFile`. This is a minor addition to the existing `ParsedFile` struct in `parser/mod.rs`, not a separate file.
  - `Framework` enum extension: the adapter scaffold emits a `todo!()` for the `framework()` method because adding a new variant to the `Framework` enum in `infergen-core` is a breaking change for community plugins. The guide must document this clearly and recommend submitting a PR or using a sentinel approach.
  - No runtime (TypeScript) plugin scaffolding is in scope for E7.3; the TypeScript SDK is a seed (E0.1-level). Community TS providers are a future concern.
