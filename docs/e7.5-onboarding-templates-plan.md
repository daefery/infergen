## Summary

E7.5 delivers three things:

1. **Per-stack scaffold** — after `infergen init`, automatically write a `.infergen/catalog.yaml`
   populated with 2–3 example events appropriate for the detected stack, so developers
   have a concrete starting point without running `infergen scan` first.
2. **Example projects** — minimal but runnable project trees in `/examples/` covering
   Next.js and Express, each showing the full init → catalog → snippet flow.
3. **Quickstart doc** — `docs/quickstart.md`, a < 5 min getting-started guide fulfilling
   PRD §12 Time-to-first-event metric.

The `--no-example` flag on `infergen init` lets power users skip scaffolding.

### 10. `crates/infergen-cli/src/commands/init.rs` (modify)

**Purpose:** After writing the config, scaffold `.infergen/catalog.yaml` with example events
(unless `--no-example`) and print stack-specific quickstart steps.

**Score: 97%**

**New `scaffold_example` function:**

```rust
fn scaffold_example(
    dir: &Path,
    result: &DetectionResult,
    config: &Config,
) -> anyhow::Result<()>
```

Logic:
1. `let template = infergen_core::template_for_frameworks(&result.frameworks);`
2. Derive catalog path: `dir.join(&config.catalog)` (default `.infergen/catalog.yaml`).
3. If catalog already exists: print `"infergen: example catalog already exists, skipping"` and return Ok.
4. Create parent directories: `fs::create_dir_all(catalog_path.parent().unwrap())?`
5. Write `template.example_catalog_yaml` bytes to `catalog_path`.
6. Print `"infergen: wrote example catalog to {catalog_path}"`.

**Updated `run` function** (after step 5 — print summary):

```rust
// 6. Scaffold example unless suppressed.
if !args.no_example {
    scaffold_example(dir, &result, &config)?;
}

// 7. Print stack-specific quickstart.
print_quickstart(&result.frameworks);
```

**New `print_quickstart` function:**

```rust
fn print_quickstart(frameworks: &[Framework]) {
    let template = infergen_core::template_for_frameworks(frameworks);
    println!("\nQuickstart ({})\n", template.stack_name);
    for (i, step) in template.quickstart_steps.iter().enumerate() {
        println!("  {}. {}", i + 1, step);
    }
    println!();
}
```

**Edge cases:**
- Catalog parent dir may not exist → `create_dir_all` handles it.
- Catalog already exists → skip silently (don't overwrite user edits).
- `--force` does NOT force-overwrite the catalog (only the config file).
- `dir` may be non-current-directory → all paths are `dir.join(...)`.

**Dependencies:** `infergen_core::templates`, `std::fs`, existing `DetectionResult`/`Config`.

**Tests (`#[cfg(test)]` in same file):**

**Structure:**

```markdown
# Infergen Quickstart

Get from zero to a tracked event in under 5 minutes.

## Prerequisites
- Rust toolchain (or download a pre-built binary when available)
- A project directory (Next.js, Express, Django, Go, Ruby on Rails, etc.)

## Step 1 — Install

    cargo install infergen

## Step 2 — Init

    cd my-project
    infergen init

Infergen detects your stack, writes `infergen.config.json`, and scaffolds
`.infergen/catalog.yaml` with example events.

## Step 3 — Scan

    infergen scan

Adds detected events to the catalog as `proposed`.

## Step 4 — Review

    infergen review list
    infergen review approve --all

## Step 5 — Generate

    infergen generate

Emits a typed SDK at `infergen.generated.ts` (or language equivalent).

## Step 6 — Track

    import { track } from "./infergen.generated";
    track.userSignupCompleted({ method: "google" });

## CI

Add `infergen scan --check` to your CI pipeline to catch untracked moments.

## Stacks

Links to per-stack examples: examples/nextjs-app, examples/express-app.
```

**Dependencies:** None (docs).

**Tests:** None (static documentation).

## Open Questions / Risks

- `include_str!` paths are relative to `src/templates.rs`; the `crates/infergen-core/templates/`
  directory must exist at compile time. Build will fail at the `include_str!` macro if any
  template file is missing — this is intentional and desirable (fail fast at compile time).
- The `--no-example` flag changes init's side effects but not its exit code — no breaking
  change for scripts that test exit codes.
- Catalog IDs in example files use `evt_example_000N` (not real stable IDs). This is safe
  because example catalogs are `proposed`; re-scan will assign real IDs.
