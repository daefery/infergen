## Open Questions / Risks

1. **Convention vs. case** — `NamingConfig` has two fields: `convention` (structural template like `"entity.action.state"`) and `case` (format like `"snake_case"`). For E1.3, linting focuses on the `case` field (format compliance). Structural `convention` linting (checking entity/action/state parts semantically) requires understanding of name composition and is deferred to E1.3's future extension or E1.4+. This scope is explicit in the plan.
2. **`is_valid_case` vs. structural checks** — consecutive underscores and leading/trailing underscores are structural violations, not case violations. `is_valid_case` for `snake_case` only validates character set (`[a-z0-9_]`), not structure. This prevents redundant violation reports (one violation per root cause).
3. **Ignored entries** — the linter skips entries with `EventStatus::Ignored` since they are intentionally excluded from tracking. `Proposed` and `Approved` entries are both linted.
4. **Empty catalog** — `lint_catalog` on an empty catalog returns an empty `Vec<LintViolation>`. Not an error.
5. **Unknown case in config** — if `naming.case` is an unrecognised string, `ConventionCase::from_config` defaults to `SnakeCase`. This matches the existing `Namer` default behavior.
6. **`apply_case` with empty token slice** — returns `""`. Callers should not rely on suggestions for entries with `EmptyName` violations (no suggestion is set).

### 1. `crates/infergen-core/src/linter/mod.rs`

**Path:** `crates/infergen-core/src/linter/mod.rs`

**Purpose:** Naming convention engine — validate catalog event names against configured case format and emit structured violations with auto-fix suggestions.

**Dependencies:** `crate::config::NamingConfig`, `crate::namer::split_identifier`, `infergen_types::{Catalog, EventStatus}`.

#### Types

```rust
use std::fmt;
use infergen_types::{Catalog, EventStatus};
use crate::{config::NamingConfig, namer::split_identifier};

/// Supported event name case formats.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ConventionCase {
    /// All lowercase, words separated by underscores: `user_signed_in`.
    SnakeCase,
    /// First word lowercase, subsequent words capitalised: `userSignedIn`.
    CamelCase,
    /// Every word capitalised, no separator: `UserSignedIn`.
    PascalCase,
}

impl ConventionCase {
    /// Parse `naming.case` from a [`NamingConfig`].
    ///
    /// Recognised values: `"snake_case"`, `"camelCase"`, `"PascalCase"` /
    /// `"pascal_case"`. All other values default to [`ConventionCase::SnakeCase`].
    pub fn from_config(naming: &NamingConfig) -> Self {
        match naming.case.as_str() {
            "camelCase" => Self::CamelCase,
            "PascalCase" | "pascal_case" => Self::PascalCase,
            _ => Self::SnakeCase,
        }
    }
}

impl fmt::Display for ConventionCase {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::SnakeCase  => write!(f, "snake_case"),
            Self::CamelCase  => write!(f, "camelCase"),
            Self::PascalCase => write!(f, "PascalCase"),
        }
    }
}

/// The specific rule that was violated.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum LintRule {
    /// Name characters do not match the configured case format.
    CaseViolation,
    /// Name is an empty string.
    EmptyName,
    /// Name contains two or more consecutive underscores.
    ConsecutiveUnderscores,
    /// Name starts or ends with an underscore.
    LeadingOrTrailingUnderscore,
}

/// A single naming convention violation for one catalog entry.
#[derive(Debug, Clone, PartialEq)]
pub struct LintViolation {
    /// Stable ID of the offending catalog entry (`evt_{016hex}`).
    pub event_id: String,
    /// Current name of the offending entry.
    pub event_name: String,
    /// Which rule was violated.
    pub rule: LintRule,
    /// Human-readable description of the violation.
    pub message: String,
    /// Auto-fix candidate name. `None` for `EmptyName` (can't auto-fix).
    pub suggestion: Option<String>,
}
```

#### `pub fn lint_catalog(catalog: &Catalog, naming: &NamingConfig) -> Vec<LintViolation>`

Lint all non-`Ignored` catalog entries.

Algorithm:
```
let case = ConventionCase::from_config(naming);
for each entry in catalog.events:
    if entry.status == EventStatus::Ignored → skip
    let name = &entry.name

    if name.is_empty():
        push EmptyName violation (suggestion = None)
        continue  ← no further checks on empty name

    if name.contains("__"):
        suggestion = name.split('_').filter(!empty).collect().join("_")
        push ConsecutiveUnderscores violation

    if name.starts_with('_') || name.ends_with('_'):
        suggestion = name.trim_matches('_')
        push LeadingOrTrailingUnderscore violation

    if !is_valid_case(name, &case):
        suggestion = to_convention_name(name, case.clone())
        push CaseViolation violation
```

Multiple violations per entry are allowed (a name can have both structural issues AND a case issue).

#### `pub fn is_valid_case(name: &str, case: &ConventionCase) -> bool`

| Case | Validation rule |
|------|----------------|
| `SnakeCase` | All chars in `[a-z0-9_]`. Empty string → true (structural rules catch other issues). |
| `CamelCase` | All chars alphanumeric. First char is ASCII lowercase. |
| `PascalCase` | All chars alphanumeric. First char is ASCII uppercase. |

Note: `is_valid_case` for `SnakeCase` does NOT check consecutive underscores or leading/trailing underscores — those are separate `LintRule` variants. This ensures one root-cause per violation.

```rust
pub fn is_valid_case(name: &str, case: &ConventionCase) -> bool {
    if name.is_empty() { return true; }  // EmptyName rule handles this separately
    match case {
        ConventionCase::SnakeCase => {
            name.chars().all(|c| c.is_ascii_lowercase() || c == '_' || c.is_ascii_digit())
        }
        ConventionCase::CamelCase => {
            name.chars().next().is_some_and(|c| c.is_ascii_lowercase())
                && name.chars().all(|c| c.is_alphanumeric())
        }
        ConventionCase::PascalCase => {
            name.chars().next().is_some_and(|c| c.is_ascii_uppercase())
                && name.chars().all(|c| c.is_alphanumeric())
        }
    }
}
```

#### `pub fn apply_case(tokens: &[&str], case: ConventionCase) -> String`

Format a slice of lowercase tokens to the target case.

```rust
pub fn apply_case(tokens: &[&str], case: ConventionCase) -> String {
    match case {
        ConventionCase::SnakeCase => tokens.join("_"),
        ConventionCase::CamelCase => {
            let mut out = String::new();
            for (i, token) in tokens.iter().enumerate() {
                if i == 0 {
                    out.push_str(token);
                } else {
                    let mut chars = token.chars();
                    if let Some(first) = chars.next() {
                        // to_uppercase returns an iterator; use next() for ASCII
                        for c in first.to_uppercase() { out.push(c); }
                        out.push_str(chars.as_str());
                    }
                }
            }
            out
        }
        ConventionCase::PascalCase => {
            tokens.iter().map(|token| {
                let mut chars = token.chars();
                match chars.next() {
                    None => String::new(),
                    Some(first) => {
                        let mut s = String::new();
                        for c in first.to_uppercase() { s.push(c); }
                        s.push_str(chars.as_str());
                        s
                    }
                }
            }).collect()
        }
    }
}
```

Edge cases:
- Empty slice → `""` for all cases.
- `["a"]` → `"a"` (snake), `"a"` (camel), `"A"` (pascal).

#### `pub fn to_convention_name(name: &str, case: ConventionCase) -> String`

```rust
pub fn to_convention_name(name: &str, case: ConventionCase) -> String {
    let tokens = split_identifier(name);
    let refs: Vec<&str> = tokens.iter().map(String::as_str).collect();
    apply_case(&refs, case)
}
```
