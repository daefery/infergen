## Overview

Extends Infergen's scan engine with Go support: a text-based Go source parser (the Rust codebase cannot link `go/ast` natively, so we use line-level pattern matching — the same strategy E5.1 used for Python), adapters for Gin, Echo, and net/http, and a Go codegen that emits an idiomatic Go package with typed structs, track functions, provider interface, and a basic runtime.

## Open Questions / Risks

- **True AST vs text:** Go's `go/ast` is not available from Rust. Text/line-based pattern matching catches common idioms (gin/echo router registration, `http.HandleFunc`) but will miss indirect registrations (e.g. routes built in a loop from a slice). This is the same trade-off as E5.1 Python. Confidence scores reflect this: path-based 0.90, pattern-match-based 0.75.
- **`regex` crate:** Not adding it; string `.lines()` + `.contains()` + manual extraction is sufficient and avoids a new dep.
- **`any` vs `interface{}`:** Go 1.18+ uses `any`; generated code targets Go 1.18+ (consistent with the ecosystem). Codegen emits `any`.

fn detect_go(root: &Path, result: &mut DetectionResult) {
    let go_mod = root.join("go.mod");
    if !go_mod.is_file() {
        return;
    }
    result.add_language(Language::Go);

use crate::{Result, Error, detect::Language};

/// Accepted Go source extensions.
const GO_EXTS: &[&str] = &["go"];

pub struct GoParser;

impl LanguageParser for GoParser {
    fn parse(&self, path: &Path, source: &str) -> Result<ParsedFile> {
        let ext = path.extension().and_then(|e| e.to_str()).unwrap_or("");
        if !GO_EXTS.contains(&ext) {
            return Err(Error::NotImplemented("GoParser: unsupported extension"));
        }
        let diagnostics = collect_diagnostics(source);
        Ok(ParsedFile {
            path: path.to_owned(),
            lang: Language::Go,
            source: source.to_owned(),
            diagnostics,
        })
    }
}

`collect_diagnostics(source: &str) -> Vec<Diagnostic>`: scans for unmatched braces (best-effort, non-fatal). Returns empty vec for well-formed source. Never propagates as Err — guarantees error-tolerant contract.

- Empty source → valid, zero diagnostics.
- Source with syntax errors → still returns `Ok(ParsedFile)` with diagnostics.
- `_test.go` files → accepted (valid Go).

### File 4 — `crates/infergen-core/src/adapter/gin.rs`

**Path:** `crates/infergen-core/src/adapter/gin.rs`  
**Purpose:** Detect Gin routes and handlers from Go source by line-pattern matching.  
**Dependencies:** `adapter/mod.rs` (Adapter trait, ProposedEvent, EventKind), `detect::Framework`, `namer`, `parser::ParsedFile`, `detect::Language`.

**Exported symbols:**
```rust
pub struct GinAdapter { pub project_root: PathBuf }
impl GinAdapter { pub fn new(project_root: impl Into<PathBuf>) -> Self }
impl Adapter for GinAdapter
```

**Detailed spec — detection strategy:**

Gin route registrations are lines matching `.<METHOD>("<path>", <handler>)` where `<METHOD>` ∈ {GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD, Any}. The receiver is typically `r`, `router`, `g`, `v1`, etc. — we match by the HTTP verb suffix only.

```rust
/// HTTP methods Gin recognises on a router group or engine.
const GIN_METHODS: &[&str] = &["GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS", "HEAD", "Any"];

fn extract_gin_routes(source: &str) -> Vec<(String, String, String)> {
    // Returns Vec<(http_method, route_path, handler_name)>
    let mut routes = Vec::new();
    for line in source.lines() {
        let trimmed = line.trim();
        for method in GIN_METHODS {
            // Match: .GET("/path", handlerFn) or .GET("/path", HandlerFn)
            let needle = format!(".{}(", method);
            if let Some(pos) = trimmed.find(needle.as_str()) {
                let after = &trimmed[pos + needle.len()..];
                if let Some(path) = extract_first_quoted(after) {
                    let handler = extract_handler_name(after);
                    routes.push((method.to_lowercase(), path, handler));
                }
            }
        }
    }
    routes
}

fn extract_first_quoted(s: &str) -> Option<String> {
    // Find the first `"..."` or `` `...` `` in `s`.
    let open = s.find('"').or_else(|| s.find('`'))?;
    let delim = s.chars().nth(open)?;
    let inner = &s[open + 1..];
    let close = inner.find(delim)?;
    Some(inner[..close].to_owned())
}

fn extract_handler_name(after_paren: &str) -> String {
    // Second argument after the path: `"/path", handlerName)` → "handlerName"
    // or `"/path", pkg.HandlerName)` → last segment after `.`
    let comma = match after_paren.find(',') {
        Some(i) => i,
        None => return String::new(),
    };
    let rest = after_paren[comma + 1..].trim();
    let end = rest.find(|c: char| c == ')' || c == ',').unwrap_or(rest.len());
    let raw = rest[..end].trim();
    // Strip package prefix: "controllers.Users" → "Users"
    raw.rsplit('.').next().unwrap_or(raw).trim().to_owned()
}
```

`Adapter::analyze` implementation:
1. Return empty if `file.lang != Language::Go`.
2. Call `file.with_go_source(|src| extract_gin_routes(src))` → list of `(method, path, handler)`.
3. For each route: derive event name via `Namer::derive(&NameSignals { route: Some(&path), handler_name: Some(&handler), kind: EventKind::ApiCall, component_name: None })`.
4. Push `ProposedEvent::new(name, EventKind::ApiCall, &file.path, 0.75).with_prop("endpoint", Some("string")).with_prop("method", Some("string")).with_adapter("gin")`.
5. Enrich properties via `property::enrich_hints`.

**Edge cases:**
- No Gin routes in file → empty vec (infallible).
- Route with dynamic segment (`/users/:id`) → path kept as-is (namer strips `:` tokens).
- Handler is an anonymous function (`func(c *gin.Context)`) → handler name empty → namer falls back to route for entity.
- Same route registered twice → both emitted (up to caller to dedup at catalog level).

**Tests (inline `#[cfg(test)]`):**

3. `gin_adapter_detects_post_route` — `r.POST("/users", CreateUser)` → one ApiCall, method "post" in name.
4. `gin_adapter_detects_dynamic_route` — `r.GET("/users/:id", GetUser)` → event name contains "users".

### File 6 — `crates/infergen-core/src/adapter/nethttp.rs`

**Path:** `crates/infergen-core/src/adapter/nethttp.rs`  
**Purpose:** Detect `http.HandleFunc` and `http.Handle` route registrations from Go source; also detect handler function signatures.  
**Dependencies:** Same as gin.rs.

**Exported symbols:**
```rust
pub struct NetHttpAdapter { pub project_root: PathBuf }
impl NetHttpAdapter { pub fn new(project_root: impl Into<PathBuf>) -> Self }
impl Adapter for NetHttpAdapter
```

**Detailed spec:**

Three detection passes on each file:

**Pass 1 — explicit registrations:**
Pattern: `http.HandleFunc("/path", handler)` or `mux.HandleFunc("/path", handler)` or `http.Handle("/path", handler)`.
```rust
fn extract_http_handlefuncs(source: &str) -> Vec<(String, String)> {
    // Returns Vec<(route_path, handler_name)>
    let mut routes = Vec::new();
    for line in source.lines() {
        let t = line.trim();
        for needle in &["http.HandleFunc(", "http.Handle(", ".HandleFunc(", ".Handle("] {
            if let Some(pos) = t.find(needle) {
                let after = &t[pos + needle.len()..];
                if let Some(path) = extract_first_quoted(after) {
                    let handler = extract_handler_name(after);
                    routes.push((path, handler));
                }
            }
        }
    }
    routes
}
```

**Pass 2 — handler function signatures:**
Pattern: `func XxxHandler(w http.ResponseWriter, r *http.Request)` or `func (h *XxxHandler) ServeHTTP(...)`.
```rust
fn extract_handler_funcs(source: &str) -> Vec<String> {
    let mut funcs = Vec::new();
    for line in source.lines() {
        let t = line.trim();
        if t.starts_with("func ") && t.contains("http.ResponseWriter") {
            // Extract function name: "func FooHandler(...)" → "FooHandler"
            if let Some(name) = extract_func_name(t) {
                funcs.push(name);
            }
        }
    }
    funcs
}

fn extract_func_name(line: &str) -> Option<String> {
    // "func FuncName(" → "FuncName"
    // "func (r *Recv) MethodName(" → "MethodName"
    let after_func = line.strip_prefix("func ")?.trim();
    if after_func.starts_with('(') {
        // Method: "(r *Recv) MethodName("
        let close = after_func.find(')')?;
        let rest = after_func[close + 1..].trim();
        let end = rest.find('(')?;
        Some(rest[..end].trim().to_owned())
    } else {
        let end = after_func.find('(')?;
        Some(after_func[..end].trim().to_owned())
    }
}
```

**analyze logic:**
1. Return empty if `file.lang != Language::Go`.
2. Pass 1: `extract_http_handlefuncs` → for each `(path, handler)`: derive event via Namer with route + handler, emit ApiCall, confidence 0.80.
3. Pass 2: `extract_handler_funcs` → for each function name not already covered by Pass 1: derive event via Namer (handler_name only, no route), emit ApiCall, confidence 0.55. Skip names like `ServeHTTP` that are generic.
4. Dedup: if same path appeared in Pass 1 and handler also in Pass 2, don't double-emit.
5. Enrich props, set `adapter = "net-http"`.

**Edge cases:**
- `http.Handle` with a `http.StripPrefix` or handler object → handler name extraction may be empty → Namer falls back.
- `func init()`  or `func main()` accidentally matching `http.ResponseWriter` in body → Pass 2 only triggers on lines starting with `func `, so top-level function declarations only.

**Tests (inline):**
1. `nethttp_returns_empty_for_non_go_file`
2. `nethttp_detects_http_handlefunc` — `http.HandleFunc("/hello", HelloHandler)` → ApiCall.
3. `nethttp_detects_mux_handlefunc` — `mux.HandleFunc("/api", handler)` → ApiCall.
4. `nethttp_detects_handler_func_signature` — `func HealthHandler(w http.ResponseWriter, r *http.Request)` → ApiCall (confidence 0.55).

type ConsentState string

const (
    ConsentUnknown ConsentState = "unknown"
    ConsentGranted ConsentState = "granted"
    ConsentDenied  ConsentState = "denied"
)

type RedactFn func(eventName, propName string, value any) any

func SetConsent(state ConsentState)  { _mu.Lock(); _consentState = state; _mu.Unlock() }
func OptIn()                         { SetConsent(ConsentGranted) }
func OptOut()                        { SetConsent(ConsentDenied) }

Mutex `_mu sync.RWMutex` included for thread safety (Go idiomatic).

**Dispatch function:**
```go
func _dispatch(name string, props map[string]any) {
    _mu.RLock()
    consent := _consentState
    redact := _redactFn
    providers := _providers
    _mu.RUnlock()

    if consent == ConsentDenied {
        return
    }
    if redact != nil {
        for k, v := range props {
            props[k] = redact(name, k, v)
        }
    }
    for _, p := range providers {
        p.Track(name, props)
    }
}
```

**Imports** emitted at top (always include, Go compiler prunes unused — actually Go won't compile with unused imports, so only include what the generated file uses):
- `"encoding/json"` — always (used in track fns for marshal/unmarshal).
- `"sync"` — always (mutex for consent/region).

**Idempotency:** Events sorted alphabetically. No timestamps. Same catalog → same output.

**Edge cases:**
- Empty catalog → emits boilerplate (interfaces, runtime, empty const block, `var Track = struct{}{}`) with no event sections.
- `Ignored` events → excluded.
- `Proposed` events → excluded unless `config.include_proposed`.
- Properties sorted alphabetically within each struct.
- PII property → add comment `// pii:"true"` in struct field.
- `package_name` default `"infergen"`.
