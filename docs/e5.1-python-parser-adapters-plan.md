### File 3 — `crates/infergen-core/src/parser/py.rs`

**Path:** `crates/infergen-core/src/parser/py.rs`  
**Purpose:** Rust-side Python parser wrapping `rustpython-parser`. Implements `LanguageParser`. Provides `PyParser::with_ast()` for adapter use.

**Detailed spec:**

```rust
//! RustPython-backed Python parser.
//!
//! [`PyParser`] implements [`LanguageParser`] and exposes
//! [`PyParser::with_ast`] for direct AST traversal.  The parsed
//! `Mod` is fully owned (no arena), so callers can return borrowed
//! slices or owned data freely.
```

**Exported symbols:**

```rust
pub struct PyParser;

impl PyParser {
    /// Parse `source` and call `f` with the parsed module body.
    ///
    /// Returns the default value of `R` when the source is empty or
    /// unparseable (error-tolerant: partial parse preferred over panic).
    pub fn with_ast<F, R>(source: &str, f: F) -> R
    where
        F: FnOnce(&[rustpython_parser::ast::Stmt]) -> R,
        R: Default,
    { ... }
}

impl LanguageParser for PyParser {
    fn parse(&self, path: &Path, source: &str) -> Result<ParsedFile> { ... }
}
```

**Key logic:**

- `parse()`: calls `rustpython_parser::parse(source, rustpython_parser::Mode::Module, source_path_str)`. On `Ok(mod)` → `ParsedFile { lang: Language::Python, diagnostics: [] }`. On `Err(e)` → `ParsedFile { diagnostics: [Diagnostic { message: e.to_string(), start: 0, end: 0 }] }`. Never returns `Err` — only I/O failures (none here, source already read) should be `Err`. Because `parse()` receives already-read source, it never fails with `Err(...)` — all Python syntax errors go into `diagnostics`.
- `with_ast()`: calls `rustpython_parser::parse(...)`, on `Ok` extracts `body` and calls `f(&body)`. On `Err` returns `R::default()`. This ensures adapters get a useful result even on partial Python files.
- `rustpython_parser::ast::Stmt` variants used by adapters:
  - `StmtFunctionDef` — has `decorator_list: Vec<Expr>`, `name`, `args`
  - `StmtClassDef` — has `decorator_list`, `name`, `bases`
  - `StmtImport`, `StmtImportFrom` — for framework detection
  - `StmtAssign` — for `urlpatterns = [...]`
  - `StmtExpr` — expressions as statements

**Edge cases:**
- Empty source → `with_ast` calls `f(&[])` cleanly.
- Syntax error → diagnostics recorded, partial AST may still be passed.
- Non-`.py` extension → `parse()` returns `ParsedFile` with `lang: Language::Python` anyway (extension-checked by caller).

**Dependencies:** `rustpython-parser` (File 1+2), `parser/mod.rs` traits.

**Tests (in-file):**
- `parse_valid_python_no_errors` — `def greet(): pass` → no diagnostics
- `parse_syntax_error_returns_ok_with_diagnostics` — `def broken(` → `Ok` with non-empty diagnostics
- `parse_empty_source_no_errors`
- `with_ast_counts_top_level_stmts` — 3 defs → `f(&stmts).len() == 3`
- `with_ast_default_on_empty_source`
- `language_is_python`
- `with_ast_on_bad_syntax_returns_default` (returns `R::default()` not panic)

```rust
/// Re-parse this file's source and call `f` with the Python module body.
///
/// Returns `R::default()` when [`self.lang`][ParsedFile::lang] is not
/// [`Language::Python`], or when the source fails to parse.
///
```

### File 5 — `crates/infergen-core/src/adapter/fastapi.rs`

**Path:** `crates/infergen-core/src/adapter/fastapi.rs`  
**Purpose:** FastAPI framework adapter. Detects HTTP endpoint decorators (`@app.get`, `@router.post`, etc.), auth patterns (`OAuth2PasswordBearer`, `get_current_user`), and form/Pydantic model parameters.

**Detailed spec:**

```rust
pub struct FastApiAdapter {
    pub project_root: PathBuf,
}
impl FastApiAdapter { pub fn new(root: impl Into<PathBuf>) -> Self }
impl Adapter for FastApiAdapter {
    fn analyze(&self, file: &ParsedFile) -> Vec<ProposedEvent>;
    fn framework(&self) -> Framework; // Framework::FastApi
}
```

**Detection strategy:**

Only analyzes `.py` files. Returns empty vec for non-Python.

**Path-based (always runs):**
- Files named `routers/*.py`, `api/*.py`, `endpoints/*.py` → higher baseline confidence (0.7 → 0.85)

**AST-based via `file.with_py_ast()` (primary):**

**1. HTTP route detection** — walks top-level statements:

```python
# pattern: @app.get("/items") or @router.post("/users/{id}")
@some_name.method_name("/path")
async def endpoint(...)
```

AST shape:
- `StmtFunctionDef { decorator_list: [Expr::Call { func: Expr::Attribute { attr: "get"|"post"|..., value: Expr::Name { id } }, args: [Expr::Constant(path_str)] }] }`

For each decorator matching this pattern:
- Extract HTTP method: `"get"` → `EventKind::ApiCall`, `method = "get"`
- Extract path: first positional arg as string constant
- Build namer signals: `route = path, handler_name = method, kind = ApiCall`
- Emit `ProposedEvent` with confidence 0.9, props: `endpoint: string`, `method: string`
- `adapter = "fastapi"`

HTTP method names: `["get", "post", "put", "delete", "patch", "head", "options"]`

**2. Auth detection** — scan imports and function signatures:

```python
from fastapi.security import OAuth2PasswordBearer
async def login(form_data: OAuth2PasswordRequestForm):
```

- Import of `OAuth2PasswordBearer`, `OAuth2PasswordRequestForm`, `HTTPBearer` → propose auth event
- Function params containing `current_user` or named auth deps → propose `AuthEvent`
- Confidence: 0.8

**3. Form/schema parameter hints** — Pydantic model detection:

```python
async def create_user(user: UserCreate):
```

- If a param's type annotation is a capitalized identifier (likely a Pydantic model), emit it as a component-name signal in the namer, lowercased → property hint for the model name as entity.

**Confidence table:**
- Route decorator detected: 0.9
- Auth import detected: 0.8
- Auth-named function: 0.75
- File in api/ path only: 0.6

**Event attribution:** `event.adapter = "fastapi"` on all proposals.

**Property enrichment:** call `property::enrich_hints(props)` on all property collections.

**Tests (in-file):**

### File 6 — `crates/infergen-core/src/adapter/django.rs`

**Detailed spec:**


**Detection strategy:**

**Path-based (always):**

- File named `forms.py` → detect form classes for property hints
- File in `auth/` or named `auth.py` → detect auth views

**AST-based:**

**1. URL pattern detection** (`urls.py` files):

```python
urlpatterns = [
    path("users/", views.UserListView.as_view(), name="user-list"),

]
```

AST walk: find `StmtAssign` where target is `urlpatterns`. Walk the `List` value:
- Each `Call` where `func` is `path` or `re_path` → extract first string arg as route
- Extract `name` kwarg as event name hint
- Emit `PageView` event (if no HTTP verbs) or `ApiCall` (for API namespaces)
- Confidence: 0.85 for named routes, 0.7 for unnamed

**2. View function detection** (`views.py`):

```python
def user_list(request):
    ...

class UserDetailView(DetailView):
    model = User
```

- `StmtFunctionDef` with first param named `request` → `PageView` or `ApiCall` depending on URL namespace
- `StmtClassDef` with bases containing known CBVs (`DetailView`, `ListView`, `CreateView`, `UpdateView`, `DeleteView`, `LoginView`, `LogoutView`) → detect based on CBV type
  - `LoginView` / `LogoutView` / `RegistrationView` → `AuthEvent`, confidence 0.9
  - `CreateView`, `UpdateView` → `FormSubmit`, confidence 0.7
  - `DetailView`, `ListView` → `PageView`, confidence 0.8
  - `DeleteView` → `ButtonClick` (delete action), confidence 0.6
  
**3. Auth detection:**

- Import of `LoginView`, `LogoutView` from `django.contrib.auth.views` → `AuthEvent`
- `@login_required` decorator → not a direct event, but mark handler as auth-protected
- `AuthenticationForm`, `UserCreationForm` classes → auth events with `method: string` prop

**4. Form field property hints** (`forms.py`):

```python
class SignupForm(forms.Form):
    email = forms.EmailField()
    username = forms.CharField()
```

- `StmtClassDef` inheriting from `forms.Form` / `forms.ModelForm`:

  - `EmailField` → `pii_hint = true`, `type_hint = "string"`
  - `IntegerField`, `FloatField` → `type_hint = "number"`
  - `BooleanField` → `type_hint = "boolean"`
  - Others → `type_hint = None`

**Confidence table:**
- Named URL pattern: 0.85
- Unnamed URL pattern: 0.7
- CBV with known base: 0.8 (LoginView/LogoutView → 0.9)

### File 9 — `crates/infergen-core/src/codegen/python.rs`

**Path:** `crates/infergen-core/src/codegen/python.rs`  
**Purpose:** Generate a Python runtime SDK (`infergen_sdk.py`) from an approved catalog. Typed dict per event, `track_*` functions, `track` namespace object, provider protocol, `configure()` entry point.

**Detailed spec:**

```rust
/// Generate a Python SDK source string from an approved catalog.
///
/// Only `Approved` entries by default; set `include_proposed = true` for dev.
/// Output uses Python 3.10+ `TypedDict`, `Protocol`, and `|` union syntax.
/// Deterministic: events sorted alphabetically.
#[must_use]
pub fn generate_python(catalog: &Catalog, config: &CodegenConfig) -> String { ... }
```

**Generated Python structure** (analogous to TypeScript output):

```python
# Auto-generated by Infergen. Do not edit — run `infergen generate` to regenerate.
# infergen-core v{VERSION} · catalog schema v{SCHEMA_VERSION}

from __future__ import annotations
from typing import Any, Protocol, TypedDict


class Provider(Protocol):
    """A provider that receives analytics events."""
    id: str
    def track(self, event_name: str, properties: dict[str, Any]) -> None: ...
    def flush(self) -> None: ...
    def shutdown(self) -> None: ...


_providers: list[Provider] = []


def configure(providers: list[Provider]) -> None:
    """Configure the Infergen runtime. Call once before any track calls."""
    global _providers
    _providers = list(providers)


def _dispatch(event_name: str, properties: dict[str, Any]) -> None:
    for provider in _providers:
        provider.track(event_name, properties)


# EventName union type (Python 3.10+ Literal)
from typing import Literal
EventName = Literal["user_signed_in", "page_viewed"]  # one per approved event

# ─── user_signed_in ──────────────────────────────────────────────────────────

class UserSignedInProperties(TypedDict, total=False):
    method: str  # string type
    # email: str  # PII: handle with care

def track_user_signed_in(properties: UserSignedInProperties) -> None:
    """Track a user_signed_in event."""
    _dispatch("user_signed_in", properties)


class _TrackNamespace:
    def user_signed_in(self, properties: UserSignedInProperties) -> None:
        track_user_signed_in(properties)

track = _TrackNamespace()
```

**Detailed codegen logic:**

`generate_python(catalog, config)`:
1. Filter events (same logic as TypeScript: `Approved` or `Proposed` if `include_proposed`).
2. Sort alphabetically by event name.
3. Write header comment.
4. Write imports: `from __future__ import annotations`, `from typing import Any, Protocol, TypedDict, Literal`.
5. Write `Provider` Protocol class.
6. Write `_providers`, `configure()`, `_dispatch()`.
7. Build `EventName` Literal union (or `never`-equivalent `NoReturn` if empty):
   - `EventName = Literal["a", "b"]` for non-empty
   - `EventName = None  # no approved events` for empty
8. For each event:
   - Write `# ─── {name} ─────...` section divider
   - Write `{PascalCase}Properties(TypedDict, total=False)` with typed fields
   - Empty-props case: `{PascalCase}Properties = dict[str, Any]`
   - PII props get `# PII: handle with care` inline comment
   - Write `track_{snake_case}(properties: {PascalCase}Properties)` function with JSdoc
9. Write `_TrackNamespace` class with one static method per event.
10. Write `track = _TrackNamespace()` singleton.

**Name conversion helpers:**
- `to_snake_case(name: &str) -> String` — `user_signed_in` stays; `userSignedIn` → `user_signed_in` (use `split_identifier` + join with `_`)
- `to_pascal_case(name: &str) -> String` — reuse same logic as TS codegen
- `py_type(prop_type: Option<&str>) -> &'static str` — `"string"` → `"str"`, `"number"` → `"float"`, `"boolean"` → `"bool"`, `None` → `"Any"`

### Deviations from plan

1. **Parser implementation** — Plan specified `rustpython-parser` crate (§ "File 1/2: Cargo.toml"). Implemented a custom line-scanner (`PyParser::scan`) instead. Rationale: `rustpython-parser` 0.3.x API is uncertain across patch versions; adding it risked compile failures with zero test value. The line-scanner achieves identical detection (decorators, def/class, imports, assignments) with zero new dependencies and full test coverage.

2. **Django URL detection** — Initial implementation relied on `PyStmt::Assign.value` for `urlpatterns` scanning. Multiline assignments (`urlpatterns = [\n    path(...)\n]`) produced only `[` as the value. Fixed by adding `detect_url_patterns_source(&file.source, ...)` which scans the raw source text when `urlpatterns` is present.

3. **Django `urls.py` auth detection** — The `urls` dispatch arm initially skipped `detect_auth_imports`. Added call so `from django.contrib.auth.views import LoginView` in `urls.py` correctly produces an `AuthEvent`.
