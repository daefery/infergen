## Recommendations

1. **`NPM_TOKEN` secret** — must be created in GitHub repository settings before first tag push for `publish-npm.yml` to succeed.
2. **First release** — run `./scripts/bump-version.sh 0.1.0` or `cargo release minor` to bump from `0.0.0` → `0.1.0`, then tag `v0.1.0` to trigger cargo-dist + npm publish pipelines.
3. **Catalog schema v2** — when a breaking catalog change is needed, bump `CATALOG_SCHEMA_VERSION` to `2` in `infergen-types/src/lib.rs` and add `fn migrate_v1_to_v2(catalog: Catalog) -> Result<Catalog>` in `migration.rs`.
4. **Prerelease npm tags** — for prerelease versions (e.g. `0.2.0-alpha.1`), add `--tag next` to the npm publish command in `publish-npm.yml` to avoid promoting a prerelease to `latest`.
