## Recommendations

- E3.2 (First-Party Adapters) should implement `ProviderPlugin` for PostHog, Segment, Amplitude. The `ProviderConfig.config` map carries their API keys and endpoints.
- E3.3 (Delivery Engine) should make `ProviderRegistry::track` async and add per-provider buffering + retry, replacing the current synchronous fan-out.
- Generated `trackXxx()` functions use `_providers.forEach` which is synchronous. E3.3 should update codegen to `await Promise.all(_providers.map(...))` once async dispatch is wired.
