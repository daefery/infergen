```markdown
# Infergen

One-liner + badge row (CI).

## What it does  (3 bullets)

## Features  (grouped by milestone category)
- Scan: JS/TS, Python, Go, Ruby + 12 framework adapters
- Catalog: YAML, stable IDs, PII flags, diff-friendly
- Codegen: TypeScript, Python, Go, Ruby SDKs
- Providers: Segment/Amplitude/Mixpanel/PostHog/GA4/RudderStack/DB/HTTP
- CI: drift detection, GitHub Action, watch mode
- Intelligence: LLM refinement (Ollama/Claude/OpenAI), semantic flow detection, quality loop
- DX: catalog web viewer, VS Code hints, plugin SDK, manifest export, guided `init`

## Quickstart (< 5 min to first tracked event)
cd my-project
cargo install infergen   # or: cargo build + PATH
infergen init            # detects stack, writes infergen.config.json
infergen scan            # proposes event catalog → .infergen/catalog.yaml
infergen review approve --all
infergen generate        # emits typed SDK
# import and use the SDK

## Commands

| Command | Description |
|---------|-------------|
| init    | Detect stack, write config |
| scan    | Propose event catalog from source |
| generate | Emit typed SDK from catalog |
| check   | CI drift detection |
| watch   | Live re-scan on file change |
| review  | Approve / ignore / rename events |
| view    | Open offline HTML catalog viewer |
| manifest| Export PII/compliance manifest |
| plugin  | Scaffold new provider/adapter/parser |

## Configuration

## Architecture

## Contributing

## License
```
