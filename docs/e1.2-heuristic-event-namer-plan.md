## Open Questions / Risks

1. **Naming vs. convention enforcement** — E1.2 ships snake_case `{entity}_{action}` as the only built-in convention. E1.3 adds configurable conventions and linting. E1.2 must not over-design convention machinery; leave that surface to E1.3.
2. **route_to_name_prefix migration** — This helper currently lives in `adapter/nextjs.rs`. E1.2 moves it to `namer/mod.rs` (pub) and re-imports it from nextjs.rs. Existing nextjs tests test the helper by calling it directly; after the move they test the namer's export. All assertions remain valid.
3. **Confidence values backward compat** — Two existing test assertions check confidence values: `assert_eq!(events[0].confidence, 0.9_f32)` in `analyzes_pages_router_page`. The namer's route-based `derive` returns `confidence=0.9` for route signals, preserving this. HTTP method detection keeps `0.95` (set by the adapter, not the namer's result).
4. **Adapter attribution for sub-function proposals** — `detect_auth_from_ast`, `detect_forms_from_ast`, `detect_http_methods_from_ast` all create proposals without adapter info. The adapter's `analyze` method post-processes all returned proposals to set `adapter = "nextjs"` before returning.
5. **split_identifier edge cases** — All-uppercase strings (e.g. "GET", "POST") must be treated as single lowercase tokens. Consecutive uppercase sequences (e.g. "APIRoute") split correctly: "api" + "route". Digits are not handled in v1 (E1.3 scope).

```rust
/// Input signals for name derivation.
pub struct NameSignals<'a> {
    /// PascalCase/camelCase component name, e.g. "UserProfilePage".
    pub component_name: Option<&'a str>,
    /// Route path, e.g. "/blog/[slug]" or "/api/users".
    pub route: Option<&'a str>,
    /// Handler function name, e.g. "handleSubmit", "signIn", "GET".
    pub handler_name: Option<&'a str>,
    /// Broad event category (used for action defaults and entity fallbacks).
    pub kind: crate::adapter::EventKind,
}

/// Output of name derivation.
pub struct NameResult {
    /// Derived event name in snake_case, e.g. "about_page_viewed".
    pub name: String,
    /// Naming confidence 0.0–1.0 reflecting signal richness.
    /// Route: 0.9, component: 0.85, handler: 0.75, fallback only: 0.5.
    pub confidence: f32,
}

/// Stateless heuristic event namer.
///
/// Uses a snake_case `{entity}_{action}` convention. E1.3 adds configurable
/// conventions; this namer ships the default.
#[derive(Debug, Default, Clone, Copy)]
pub struct Namer;
```

#### `impl Namer`

```rust
impl Namer {
    pub fn new() -> Self { Self }

    /// Derive a canonical event name from `signals`.
    ///
    /// Signal priority for entity: route > component_name > handler_name > kind fallback.
    /// Signal priority for action: handler_name > kind default.
    pub fn derive(&self, signals: &NameSignals<'_>) -> NameResult {
        let (entity, entity_conf) = self.derive_entity(signals);
        let (action, action_conf) = self.derive_action(signals);

        let name = if entity.is_empty() {
            action.clone()
        } else {
            format!("{entity}_{action}")
        };

        // Overall confidence: weight entity slightly higher (entity richness
        // dominates naming quality), clamp to [0.0, 1.0].
        let confidence = (entity_conf * 0.6 + action_conf * 0.4).min(1.0);

        NameResult { name, confidence }
    }

    fn derive_entity(&self, signals: &NameSignals<'_>) -> (String, f32) {
        // Route wins: most reliable entity signal.
        if let Some(route) = signals.route {
            return (route_to_name_prefix(route), 0.9);
        }

        // Component name: strip known action words, keep noun tokens.
        if let Some(comp) = signals.component_name {
            let tokens = split_identifier(comp);
            let entity_tokens: Vec<&str> = tokens.iter()
                .map(String::as_str)
                .filter(|t| !ACTION_WORDS.contains(t))
                .collect();
            if !entity_tokens.is_empty() {
                return (entity_tokens.join("_"), 0.85);
            }
        }

        // Handler name: strip handler prefixes AND action words to isolate entity.
        if let Some(handler) = signals.handler_name {
            let tokens = split_identifier(handler);
            let lc: Vec<String> = tokens.iter().map(|t| t.to_lowercase()).collect();
            let entity_tokens: Vec<&str> = lc.iter()
                .map(String::as_str)
                .filter(|t| !HANDLER_PREFIXES.contains(t) && !ACTION_WORDS.contains(t))
                .collect();
            if !entity_tokens.is_empty() {
                return (entity_tokens.join("_"), 0.7);
            }
        }

        // Kind-based fallback — low confidence since no identifier signals used.
        use crate::adapter::EventKind;
        let fallback = match signals.kind {
            EventKind::PageView  => "page",
            EventKind::FormSubmit => "form",
            EventKind::AuthEvent  => "user",
            EventKind::ApiCall    => "api",
            EventKind::Error      => "error",
        };
        (fallback.to_owned(), 0.5)
    }

    fn derive_action(&self, signals: &NameSignals<'_>) -> (String, f32) {
        // Try to detect action from handler name first.
        if let Some(handler) = signals.handler_name {
            let tokens = split_identifier(handler);
            let lc: Vec<String> = tokens.iter().map(|t| t.to_lowercase()).collect();
            let lc_refs: Vec<&str> = lc.iter().map(String::as_str).collect();

            if let Some(action) = detect_action_from_tokens(&lc_refs, signals.kind) {
                return (action, 0.9);
            }
        }

        // Kind-default action.
        use crate::adapter::EventKind;
        let action = match signals.kind {
            EventKind::PageView   => "page_viewed",
            EventKind::FormSubmit => "submitted",
            EventKind::AuthEvent  => "triggered",
            EventKind::ApiCall    => "api_called",
            EventKind::Error      => "errored",
        };
        (action.to_owned(), 0.75)
    }
}
```

#### `pub fn split_identifier(s: &str) -> Vec<String>`

Splits any code identifier into lowercase tokens.

Rules (applied in order):
1. Split on `_` and `-` (snake/kebab boundaries).
2. For each resulting part, split camelCase/PascalCase boundaries.
3. All tokens lowercased.
4. Empty tokens discarded.

camelCase boundary: uppercase char where previous char is lowercase, OR uppercase char where next char is lowercase and previous chars were all uppercase (handles "APIRoute" → ["api", "route"]).

```rust
pub fn split_identifier(s: &str) -> Vec<String> {
    if s.is_empty() { return Vec::new(); }

    let parts: Vec<&str> = s.split(|c: char| c == '_' || c == '-')
        .filter(|p| !p.is_empty())
        .collect();

    let mut tokens = Vec::new();
    for part in parts {
        tokens.extend(split_camel_case(part));
    }
    tokens
}

fn split_camel_case(s: &str) -> Vec<String> {
    if s.is_empty() { return Vec::new(); }
    let chars: Vec<char> = s.chars().collect();
    let n = chars.len();
    let mut tokens = Vec::new();
    let mut start = 0;

    for i in 1..n {
        let c = chars[i];
        if c.is_uppercase() {
            let prev_lower = chars[i - 1].is_lowercase() || chars[i - 1].is_numeric();
            let next_lower = i + 1 < n && (chars[i + 1].is_lowercase() || chars[i + 1].is_numeric());
            // Boundary: camelCase (prevLower) or start of word after acronym (nextLower)
            if prev_lower || (next_lower && i > start + 1) {
                let token: String = chars[start..i].iter().collect::<String>().to_lowercase();
                if !token.is_empty() { tokens.push(token); }
                start = i;
            }
        }
    }
    let last: String = chars[start..].iter().collect::<String>().to_lowercase();
    if !last.is_empty() { tokens.push(last); }
    tokens
}
```

Special cases verified:
- `"GET"` → no uppercase boundary triggers (all uppercase, only one group) → ["get"] ✓
- `"UserProfile"` → boundary at 'P' (prev='r' lowercase) → ["user", "profile"] ✓
- `"handleSubmit"` → boundary at 'S' (prev='e' lowercase) → ["handle", "submit"] ✓
- `"onSignIn"` → boundary at 'S' (prev='n'), boundary at 'I' (prev='n') → ["on", "sign", "in"] ✓
- `"APIRoute"` → 'R' at i=3: prev='I' uppercase, next='o' lowercase, i > start+1 → ["api", "route"] ✓

```rust
pub fn route_to_name_prefix(route: &str) -> String {
    let stripped = route.trim_start_matches('/');
    if stripped.is_empty() {
        return "home".to_owned();
    }
    stripped
        .replace('/', "_")
        .replace(['[', ']'], "")
        .replace('-', "_")
        .to_lowercase()
}
```

#### `fn detect_action_from_tokens(tokens: &[&str], kind: EventKind) -> Option<String>`

Private. Maps token lists to action strings:

- HTTP method (kind=ApiCall, single token in HTTP_METHODS): `"{method}_api_called"`
- Auth sign-in patterns: tokens contain ("sign","in") or ("log","in") or "login" or "signin" → `"signed_in"`
- Auth sign-out: ("sign","out") or ("log","out") or "logout" or "signout" → `"signed_out"`
- Auth sign-up: ("sign","up") or "signup" or "register" → `"signed_up"`
- Submit: "submit" → `"submitted"`
- Click: "click" → `"clicked"`
- Load: "load" → `"loaded"`
- Error: "error" → `"errored"`
- View: "view" → `"viewed"`
- Otherwise: `None` (caller uses kind default)
