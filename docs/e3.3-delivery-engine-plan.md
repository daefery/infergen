## Overview

E3.3 extends the TypeScript codegen to emit a `DeliveryEngine` class and
`withDelivery` helper into every generated SDK file. The delivery engine
implements the `Provider` interface (generated since E3.1) and wraps any
provider with:

- **Batching** — collect N events, flush when threshold or timer fires
- **Retry with exponential backoff** — re-attempt failed deliveries
- **Sampling** — forward only a fraction of events
- **Persistent queue** — survive browser reloads (localStorage) or process
  restarts (Node.js file) when a `persistenceKey` is set
- **Flush-on-exit** — Node `process.on("beforeExit")`, browser
  `visibilitychange` hidden

All code is pure TypeScript generated by Rust string functions — zero runtime
npm dependencies, zero changes to the Rust `ProviderPlugin`/`ProviderRegistry`
(those already live in `provider/mod.rs`).

// ─── Delivery Engine ─────────────────────────────────────────────────────────

/** Options for DeliveryEngine. */
export interface DeliveryOptions {
  /** Number of events to collect before an automatic flush. Default: 10. */
  batchSize?: number;
  /** Milliseconds between timer-driven flushes. 0 disables the timer. Default: 5000. */
  flushInterval?: number;
  /** Maximum delivery attempts per event before it is dropped. Default: 3. */
  maxRetries?: number;
  /** Base retry delay in milliseconds; doubles each attempt (exponential backoff). Default: 1000. */
  retryDelay?: number;
  /** Fraction of events to forward (0.0 = drop all, 1.0 = forward all). Default: 1.0. */
  sampleRate?: number;
  /** Maximum number of queued events. Oldest is dropped when the queue is full. Default: 1000. */
  maxQueueSize?: number;
  /**
   * Persistence key for the event queue.
   * Browser: key in localStorage. Omit for in-memory queue only.
   */
  persistenceKey?: string;
}

/** @internal Queued event payload — compact keys keep storage small. */
interface _QueuedEvent {
  n: string;
  p: Record<string, unknown>;
}

/**
 * Wraps a {@link Provider} with batching, retry, sampling, and flush-on-exit.
 *
 * @example
 * ```ts
 * configureInfergen({
 *   providers: [withDelivery(new PostHogProvider({ apiKey: "phc_..." }))],
 * });
 * ```
 */
export class DeliveryEngine implements Provider {
  readonly id: string;
  private _q: _QueuedEvent[] = [];
  private _timer: ReturnType<typeof setInterval> | null = null;
  private _flushing = false;
  private readonly _o: Required<Omit<DeliveryOptions, "persistenceKey">> & { persistenceKey?: string };

  constructor(private readonly _inner: Provider, opts: DeliveryOptions = {}) {
    this.id = `delivery:${_inner.id}`;
    this._o = {
      batchSize:     opts.batchSize     ?? 10,
      flushInterval: opts.flushInterval ?? 5000,
      maxRetries:    opts.maxRetries    ?? 3,
      retryDelay:    opts.retryDelay    ?? 1000,
      sampleRate:    opts.sampleRate    ?? 1.0,
      maxQueueSize:  opts.maxQueueSize  ?? 1000,
      persistenceKey: opts.persistenceKey,
    };
    this._load();
    this._startTimer();
    this._exitHook();
  }

  track(eventName: string, properties: Record<string, unknown>): void {
    if (this._o.sampleRate < 1.0 && Math.random() > this._o.sampleRate) return;
    if (this._q.length >= this._o.maxQueueSize) this._q.shift();
    this._q.push({ n: eventName, p: properties });
    this._save();
    if (this._q.length >= this._o.batchSize) void this._flush();
  }

  flush(): void { void this._flush(); }

  shutdown(): void { this._stopTimer(); void this._flush(); }

  private _startTimer(): void {
    if (this._o.flushInterval > 0) {
      this._timer = setInterval(() => void this._flush(), this._o.flushInterval);
      const t = this._timer as unknown as { unref?: () => void };
      if (typeof t.unref === "function") t.unref();
    }
  }

  private _stopTimer(): void {
    if (this._timer !== null) { clearInterval(this._timer); this._timer = null; }
  }

  private async _flush(): Promise<void> {
    if (this._flushing || this._q.length === 0) return;
    this._flushing = true;
    try {
      const batch = this._q.splice(0, this._o.batchSize);
      this._save();
      await Promise.all(batch.map(e => this._deliver(e)));
    } finally {
      this._flushing = false;
    }
  }

  private async _deliver(e: _QueuedEvent): Promise<void> {
    for (let i = 0; i <= this._o.maxRetries; i++) {
      try { this._inner.track(e.n, e.p); return; } catch { /* retry */ }
      if (i < this._o.maxRetries)
        await new Promise<void>(r => setTimeout(r, this._o.retryDelay * (2 ** i)));
    }
  }

  private _save(): void {
    const key = this._o.persistenceKey;
    if (!key) return;
    try {
      if (typeof localStorage !== "undefined") localStorage.setItem(key, JSON.stringify(this._q));
    } catch { /* storage quota or unavailable */ }
  }

  private _load(): void {
    const key = this._o.persistenceKey;
    if (!key) return;
    try {
      if (typeof localStorage !== "undefined") {
        const raw = localStorage.getItem(key);
        if (raw) this._q = JSON.parse(raw) as _QueuedEvent[];
      }
    } catch { /* ignore corrupt data */ }
  }

  private _exitHook(): void {
    if (typeof process !== "undefined" && typeof process.on === "function") {
      process.on("beforeExit", () => this.shutdown());
    } else if (typeof window !== "undefined") {
      window.addEventListener("visibilitychange", () => {
        if (document.visibilityState === "hidden") this.flush();
      });
    }
  }
}

/** Wrap a provider in a {@link DeliveryEngine} with optional delivery options. */
export function withDelivery(provider: Provider, opts?: DeliveryOptions): DeliveryEngine {
  return new DeliveryEngine(provider, opts);
}
```

**Implementation notes:**
- `_flushing` flag prevents concurrent flush calls from overlapping (re-entrancy guard)
- `unref()` call lets Node.js exit without waiting for the flush timer
- `2 ** i` is exponential backoff: 1000ms, 2000ms, 4000ms for `maxRetries=3`
- `Math.random() > sampleRate` samples correctly (sampleRate=0.1 → 10% forwarded)
- `Promise.all(batch.map(...))` delivers batch events concurrently
- localStorage persistence is browser-only; Node file persistence deferred to E3.4 or later

## Open questions / risks

1. **`_flushing` concurrency**: TypeScript is single-threaded in both browser and Node, so
   a simple boolean guard is sufficient. No mutex needed.
2. **Node file persistence**: deferred. Only localStorage (browser) is implemented. A
   future `persistFile?: string` option could use `node:fs` (same dynamic-import pattern
   as E3.2b's FileProvider).
3. **`flush()` return type**: existing `Provider.flush?(): void` is optional and returns void.
   `DeliveryEngine.flush()` calls `void this._flush()` to match the interface — the async
   work continues but callers get void back. For explicit await, users call
   `(engine as DeliveryEngine)._flush()` — not exposed, by design.
4. **E3.2/E3.2b not on main**: DeliveryEngine is generated independently. When E3.2 merges,
   `withDelivery(new PostHogProvider(...))` just works — no changes needed to E3.3.
