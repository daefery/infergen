## Recommendations

1. **E6.1 implementation** — Wire `OllamaRefiner: SemanticRefiner` into `FlowDetector::with_refiner` once E6.1 ships. The interface is already stable.
2. **Step ordering** — Temporal suffix heuristic covers common cases (`started`, `submitted`, `completed`). Unusual naming patterns sort lexicographically. Review step ordering after first real-project scan.
3. **Flow confidence threshold** — Currently 0.5 minimum. May want to raise to 0.65 once real-world false-positive rate is measured.
4. **`assign_flows_empty_detected` semantics** — With empty detected set, existing flows are preserved (no change). A future "clean sweep" mode could pass `clear_stale: bool` if users want auto-flows cleared on re-scan.
