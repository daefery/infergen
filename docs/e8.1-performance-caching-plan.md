## Problem Statement

The current scan pipeline (`infergen scan`) is 100% sequential:
- JS/TS scan: `for file in js_files { read → parse(OXC) → analyze }` — one CPU core, one file at a time
- Python scan: same sequential loop
- Zero caching: every scan re-reads and re-parses every file from scratch, including files unchanged since last run

On a ~100k LOC repo (~500–2000 source files), this means several seconds of wall-clock time per `infergen scan`, growing linearly with repo size. The PRD requires seconds, not minutes; incremental rescans should be sub-second.

## Solution

Two orthogonal improvements:

1. **Parallel file scan (rayon):** Replace sequential `for` loops with `rayon::par_iter`, giving near-linear speedup on multi-core machines. OXC parse allocators are thread-local; adapters are stateless and `Send + Sync`.

2. **Incremental scan cache:** On each scan, record per-file `(mtime, content_hash, proposals)`. On subsequent scans, skip re-parsing files whose mtime (and hash) are unchanged. Only files modified since last scan are re-parsed. Enables sub-second incremental rescans.

## Architecture

```
scan.rs
  ↓ load cache (.infergen/scan-cache.json)
  ↓ for each language:
      Phase 1 (sequential): check mtime for each file → split into HITS / STALE
      Phase 2 (rayon parallel): for stale files: read → hash → parse → analyze
      Phase 3 (sequential): update cache entries, merge all proposals
  ↓ save cache if dirty
  ↓ (existing) quality feedback, flow detection, catalog merge
```

## Open Questions / Risks

- OXC `Allocator` is not `Send` (arena allocator). This is safe because each rayon thread creates its own `Allocator` inside the closure — no sharing across threads. Rust will enforce this at compile time.
- FNV-1a hash has collision probability ~1/2^64 per file pair — acceptable for cache invalidation (worst case: stale proposals survive one scan; next mtime change forces re-parse).
- Python scan also parallelized via `Box<dyn Adapter + Send + Sync>` — this requires `Adapter: Send + Sync` supertrait added in File 4. All existing adapters hold only `PathBuf` so they auto-implement `Send + Sync`.
- Cache file `.infergen/scan-cache.json` should be `.gitignore`d by convention (no code enforces this — user responsibility).
- Watch mode (E4.3) does not use the cache — it already re-scans incrementally via file events. Cache is for `infergen scan` only.

```rust
//! Incremental scan cache (E8.1).
//!
//! Stores per-file `(mtime, content_hash, proposals)` in
//! `.infergen/scan-cache.json`. On the next `infergen scan`, files whose mtime
//! matches the cache entry are skipped entirely; only changed files are
//! re-parsed and re-analysed. This makes incremental rescans sub-second on
//! typical repos.

use std::{
    collections::HashMap,
    path::{Path, PathBuf},
    time::UNIX_EPOCH,
};

use serde::{Deserialize, Serialize};

use crate::adapter::ProposedEvent;

/// Version of the on-disk cache schema. Increment on breaking format changes.
pub const CACHE_VERSION: u32 = 1;

/// Per-file cache entry.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct CacheEntry {
    /// File mtime as seconds since Unix epoch.
    pub modified_secs: u64,
    /// FNV-1a hash of the file's byte content. Used as secondary validation
    /// when mtime changes but content may be identical (e.g. `touch`).
    pub content_hash: u64,
    /// Proposals produced by the last parse of this file.
    pub proposals: Vec<ProposedEvent>,
}

/// The full scan cache, serialized to `.infergen/scan-cache.json`.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct ScanCache {
    /// Schema version — always [`CACHE_VERSION`].
    pub version: u32,
    /// Entries keyed by normalized relative path (forward slashes).
    pub entries: HashMap<String, CacheEntry>,
}

impl Default for ScanCache {
    fn default() -> Self {
        ScanCache { version: CACHE_VERSION, entries: HashMap::new() }
    }
}

impl ScanCache {
    /// Look up a cache entry by normalized relative path.
    pub fn get(&self, rel_path: &str) -> Option<&CacheEntry> {
        self.entries.get(rel_path)
    }

    /// Insert or update a cache entry.
    pub fn insert(&mut self, rel_path: String, entry: CacheEntry) {
        self.entries.insert(rel_path, entry);
    }

    /// Number of cached entries.
    pub fn len(&self) -> usize {
        self.entries.len()
    }

    /// True when the cache has no entries.
    pub fn is_empty(&self) -> bool {
        self.entries.is_empty()
    }
}

/// Derive the cache file path from the catalog path.
///
/// Places the cache alongside the catalog, e.g.
/// `.infergen/catalog.yaml` → `.infergen/scan-cache.json`.
pub fn cache_path(catalog_path: &Path) -> PathBuf {
    catalog_path
        .parent()
        .unwrap_or(Path::new("."))
        .join("scan-cache.json")
}

/// Load the cache from disk. Returns an empty `ScanCache` on any error
/// (missing file, parse error, version mismatch) — cache misses are safe.
pub fn load_cache(path: &Path) -> ScanCache {
    let Ok(text) = std::fs::read_to_string(path) else {
        return ScanCache::default();
    };
    match serde_json::from_str::<ScanCache>(&text) {
        Ok(c) if c.version == CACHE_VERSION => c,
        _ => ScanCache::default(),
    }
}

/// Persist the cache to disk. Creates parent directories if needed.
///
/// # Errors
/// [`crate::Error::Io`] on file-system failure.
/// [`crate::Error::CatalogParse`] (re-used) on serialisation failure (should not happen).
pub fn save_cache(cache: &ScanCache, path: &Path) -> crate::Result<()> {
    if let Some(parent) = path.parent() {
        if !parent.as_os_str().is_empty() {
            std::fs::create_dir_all(parent)?;
        }
    }
    let text = serde_json::to_string(cache).map_err(|e| crate::Error::CatalogParse {
        path: path.to_path_buf(),
        message: e.to_string(),
    })?;
    std::fs::write(path, text)?;
    Ok(())
}

/// Get the mtime of a file as seconds since Unix epoch. Returns `None` on any
/// error (e.g. file removed between collection and metadata read).
pub fn file_mtime(path: &Path) -> Option<u64> {
    std::fs::metadata(path)
        .ok()?
        .modified()
        .ok()?
        .duration_since(UNIX_EPOCH)
        .ok()
        .map(|d| d.as_secs())
}

/// FNV-1a 64-bit hash of `data`. Non-cryptographic; used only for cache
/// invalidation where speed matters more than collision resistance.
///
/// Same algorithm as the stable-ID generation in `infergen-types`.
pub fn fnv1a_hash(data: &[u8]) -> u64 {
    const OFFSET: u64 = 14_695_981_039_346_656_037;
    const PRIME: u64 = 1_099_511_628_211;
    let mut h = OFFSET;
    for &byte in data {
        h ^= byte as u64;
        h = h.wrapping_mul(PRIME);
    }
    h
}

/// Compute the relative path from `root`, normalised with forward slashes.
///
/// Falls back to the full path if `path` is not under `root` (should not
/// happen in normal usage — all scanned files are under the project root).
pub fn normalize_path(path: &Path, root: &Path) -> String {
    path.strip_prefix(root)
        .unwrap_or(path)
        .to_string_lossy()
        .replace('\\', "/")
}
```

```rust
    let catalog_path = cwd.join(&config.catalog);
    let cache_file_path = cache_path(&catalog_path);
    let mut cache = load_cache(&cache_file_path);
    let mut cache_dirty = false;

    let js_files = collect_js_files(&cwd);
    if !js_files.is_empty() {
        println!("scan: {} JS/TS files", js_files.len());
        let adapter = NextjsAdapter::new(&cwd);

        // Phase 1: quick metadata check → separate hits from stale (sequential).
        let mut js_cached_proposals: Vec<ProposedEvent> = Vec::new();
        let mut js_stale: Vec<(PathBuf, String)> = Vec::new();

        for file_path in &js_files {
            let rel = normalize_path(file_path, &cwd);
            let mtime = file_mtime(file_path).unwrap_or(0);
            if let Some(entry) = cache.get(&rel) {
                if mtime == entry.modified_secs {
                    js_cached_proposals.extend(entry.proposals.iter().cloned());
                    continue;
                }
            }
            js_stale.push((file_path.clone(), rel));
        }

        // Phase 2: parallel re-parse of stale files.
        let stale_results: Vec<(String, u64, u64, Vec<ProposedEvent>)> = js_stale
            .par_iter()
            .map(|(file_path, rel)| {
                let Ok(source) = std::fs::read_to_string(file_path) else {
                    return (rel.clone(), 0u64, 0u64, vec![]);
                };
                let content_hash = fnv1a_hash(source.as_bytes());

                // Secondary cache check: mtime changed but content identical (e.g. `touch`).
                if let Some(cached) = cache.get(rel.as_str()) {
                    if content_hash == cached.content_hash {
                        let mtime = file_mtime(file_path).unwrap_or(0);
                        return (rel.clone(), mtime, content_hash, cached.proposals.clone());
                    }
                }

                // Full re-parse.
                let Ok(parsed) = JsParser.parse(file_path, &source) else {
                    return (rel.clone(), file_mtime(file_path).unwrap_or(0), content_hash, vec![]);
                };
                let proposals = adapter.analyze(&parsed);
                let mtime = file_mtime(file_path).unwrap_or(0);
                (rel.clone(), mtime, content_hash, proposals)
            })
            .collect();

        // Phase 3: update cache, collect proposals.
        for (rel, mtime, content_hash, proposals) in stale_results {
            if content_hash != 0 {
                cache.insert(
                    rel,
                    CacheEntry { modified_secs: mtime, content_hash, proposals: proposals.clone() },
                );
                cache_dirty = true;
            }
            all_proposals.extend(proposals);
        }
        all_proposals.extend(js_cached_proposals);
    }
```

```rust
                        return (rel.clone(), 0u64, 0u64, vec![]);
                    };
                    let content_hash = fnv1a_hash(source.as_bytes());

                    if let Some(cached) = cache.get(rel.as_str()) {
                        if content_hash == cached.content_hash {
                            let mtime = file_mtime(file_path).unwrap_or(0);
                            return (rel.clone(), mtime, content_hash, cached.proposals.clone());
                        }
                    }

                    let py_adapter: Box<dyn Adapter> = match py_framework {
                        Framework::FastApi => Box::new(FastApiAdapter::new(&cwd)),
                        Framework::Django => Box::new(DjangoAdapter::new(&cwd)),
                        Framework::Flask => Box::new(FlaskAdapter::new(&cwd)),
                        _ => Box::new(FastApiAdapter::new(&cwd)),
                    };
                    let Ok(parsed) = PyParser.parse(file_path, &source) else {
                        return (rel.clone(), file_mtime(file_path).unwrap_or(0), content_hash, vec![]);
                    };
                    let proposals = py_adapter.analyze(&parsed);
                    let mtime = file_mtime(file_path).unwrap_or(0);
                    (rel.clone(), mtime, content_hash, proposals)
                })
                .collect();

            for (rel, mtime, content_hash, proposals) in py_stale_results {
                if content_hash != 0 {
                    cache.insert(
                        rel,
                        CacheEntry { modified_secs: mtime, content_hash, proposals: proposals.clone() },
                    );
                    cache_dirty = true;
                }
                all_proposals.extend(proposals);
            }
            all_proposals.extend(py_cached_proposals);
        }
    }
```

### Edge Cases

- `cache.get(rel.as_str())` inside `par_iter`: `ScanCache` is read-only here. `&ScanCache: Sync` iff `HashMap<String, CacheEntry>: Sync` iff `CacheEntry: Sync`. All `CacheEntry` fields are `Send + Sync` primitives/strings — this compiles without explicit trait bounds.
- `content_hash == 0` guard: `fnv1a_hash(b"")` returns the FNV offset basis (non-zero), so an actual empty file would have a non-zero hash. The `0u64` sentinel is only used in the early-return branch when the file can't be read — this is correct.
- `file_mtime` double-call: called twice (once in Phase 1 for the hit check, once in Phase 2 result tuple). The Phase 1 call uses a fresh mtime; the Phase 2 call uses the mtime after parsing (small race window — acceptable for a scan cache). If the file is modified between Phase 1 and Phase 2, the updated mtime will be stored, which is correct.
- Proposals from cached hits and fresh parse are both appended to `all_proposals`. Order is non-deterministic across parallel threads, but `rescan_merge` uses stable IDs for catalog merge — order doesn't matter.
