## Scope

E3.2b delivers **three things**:

1. **TypeScript DB provider adapter classes** (Postgres, MySQL, SQLite) — generated when the provider is configured. Each adapter accepts a driver-agnostic execute/run callback so no npm dependencies appear in the generated code.

2. **TypeScript file/console adapter classes** — generated alongside DB adapters. `FileProvider` (JSONL append, Node.js `fs`) and `ConsoleProvider` (development-time logging, browser + Node).

3. **`infergen schema` CLI command** — outputs SQL CREATE TABLE + per-event typed views from the approved catalog, for the configured dialect (Postgres, MySQL, SQLite). Used to bootstrap the DB schema that the DB adapters write into.

---

## Design Decisions

### DB adapter transport — driver-agnostic callback

DB adapters are **not** bundled with a specific npm package. Instead, each adapter accepts a query callback:
- `PostgresProvider` / `MysqlProvider`: `execute: (sql: string, params: unknown[]) => Promise<void>`
- `SqliteProvider`: `run: (sql: string, params: unknown[]) => void | Promise<void>`

This is compatible with `pg`, `postgres.js`, `mysql2`, `better-sqlite3`, `node-sqlite3`, Knex, Drizzle, Prisma raw, etc. Zero generated-file npm dependencies — same principle as E3.2's `fetch()` approach.

### Events table schema (single-table, JSONB/JSON/TEXT properties)

All DB adapters write to a single `infergen_events` table with:
- `event_name TEXT` — event identifier
- `properties JSONB|JSON|TEXT` — serialized JSON of all properties
- `created_at` — timestamp

A per-event VIEW is generated for typed access (`infergen schema`). The view extracts properties into typed columns using dialect-native JSON functions.

### `infergen schema` command

New CLI subcommand. Outputs deterministic SQL to stdout (default) or a file (`--output`). Does not read `infergen.config.*` for the DB connection — only reads the catalog for event/property shapes.

### File adapter

`FileProvider` uses dynamic `import("node:fs")` inside the `track()` body so the generated SDK does not break browser bundlers that would error on a static `import fs from "fs"`. The `import()` call is deferred to call time, not parse time.

### Console adapter

`ConsoleProvider` is zero-dependency, works in browser and Node. Useful for development and debugging. No file writes, no network.

### 1. `crates/infergen-core/src/schema/mod.rs`

**Purpose:** Generate SQL CREATE TABLE and per-event CREATE VIEW statements from an approved catalog.

**Exports:**

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum SqlDialect {
    Postgres,
    Mysql,
    Sqlite,
}

impl SqlDialect {
    pub fn as_str(self) -> &'static str {
        match self {
            SqlDialect::Postgres => "postgres",
            SqlDialect::Mysql => "mysql",
            SqlDialect::Sqlite => "sqlite",
        }
    }
}

#[must_use]
pub fn generate_sql_schema(catalog: &Catalog, dialect: SqlDialect) -> String
```

**Key logic:**

1. Filter catalog events to `Approved` only. Sort alphabetically.
2. `write_schema_header` — comment header with infergen version + catalog schema version.
3. `write_events_table(out, dialect)` — generate the `infergen_events` table.
4. `write_events_index(out, dialect)` — CREATE INDEX on `event_name`.
5. For each approved event with at least one property: `write_event_view(out, entry, dialect)`.

**Per-event views:**

For `page_viewed` with property `route: string`, `user_id: string @pii`:

Postgres:
```sql
-- page_viewed
CREATE OR REPLACE VIEW "page_viewed" AS
  SELECT
    id,
    created_at,
    properties ->> 'route' AS route,
    properties ->> 'user_id' AS user_id -- @pii
  FROM "infergen_events"
  WHERE event_name = 'page_viewed';
```

MySQL:
```sql
-- page_viewed
CREATE OR REPLACE VIEW `page_viewed` AS
  SELECT
    id,
    created_at,
    JSON_UNQUOTE(JSON_EXTRACT(properties, '$.route')) AS route,
    JSON_UNQUOTE(JSON_EXTRACT(properties, '$.user_id')) AS user_id -- @pii
  FROM `infergen_events`
  WHERE event_name = 'page_viewed';
```

SQLite:
```sql
-- page_viewed
CREATE VIEW IF NOT EXISTS "page_viewed" AS
  SELECT
    id,
    created_at,
    json_extract(properties, '$.route') AS route,
    json_extract(properties, '$.user_id') AS user_id -- @pii
  FROM "infergen_events"
  WHERE event_name = 'page_viewed';
```

**Private helpers:**
- `fn write_schema_header(out: &mut String, dialect: SqlDialect)` — dialect comment + auto-gen header
- `fn write_events_table(out: &mut String, dialect: SqlDialect)` — CREATE TABLE
- `fn write_events_index(out: &mut String, dialect: SqlDialect)` — CREATE INDEX
- `fn write_event_view(out: &mut String, entry: &CatalogEntry, dialect: SqlDialect)` — CREATE VIEW per event; skipped if no properties
- `fn json_extract(col: &str, field: &str, dialect: SqlDialect) -> String` — returns the JSON extraction expression per dialect
- `fn quote_identifier(name: &str, dialect: SqlDialect) -> String` — `"name"` (Postgres/SQLite) or `` `name` `` (MySQL)

**Generated TypeScript — PostgresProvider:**

```typescript
// ─── Postgres ─────────────────────────────────────────────────────────────
// Server-side only. See `infergen schema --dialect postgres` for the table DDL.

/** Construction options for PostgresProvider. */
export interface PostgresProviderOptions {
  /** Execute a parameterized SQL statement. Compatible with pg, postgres.js, knex, Drizzle, Prisma $executeRaw, etc. */
  execute: (sql: string, params: unknown[]) => Promise<void>;
  /** Target table name. Defaults to "infergen_events". */
  table?: string;
}

/** Postgres analytics provider. Writes events to a database table — server-side only. */
export class PostgresProvider implements Provider {
  readonly id = "postgres";
  constructor(private readonly options: PostgresProviderOptions) {}
  track(eventName: string, properties: Record<string, unknown>): void {
    const table = this.options.table ?? "infergen_events";
    this.options.execute(
      `INSERT INTO "${table}" (event_name, properties, created_at) VALUES ($1, $2::jsonb, NOW())`,
      [eventName, JSON.stringify(properties)]
    ).catch(() => {});
  }
}
```

  /** Target table name. Defaults to "infergen_events". */

    ).catch(() => {});
  }

/** Construction options for FileProvider. */
export interface FileProviderOptions {
  /** Absolute or relative path to the JSONL log file. */
  path: string;
}

/** File analytics provider. Appends events as JSONL — server-side (Node.js) only. */
export class FileProvider implements Provider {
  readonly id = "file";
  constructor(private readonly options: FileProviderOptions) {}
  track(eventName: string, properties: Record<string, unknown>): void {
    import("node:fs").then(({ appendFile }) => {
      const line = JSON.stringify({ event: eventName, properties, timestamp: new Date().toISOString() }) + "\n";
      appendFile(this.options.path, line, () => {});
    }).catch(() => {});
  }
}

### 5. `crates/infergen-cli/src/commands/schema.rs`

**Purpose:** Run `infergen schema` — load catalog, generate SQL, write to file or stdout.

**Full implementation:**

```rust
//! `infergen schema` — SQL schema generation from approved catalog (E3.2b).

use anyhow::Context;
use infergen_core::{Catalog, SqlDialect, generate_sql_schema, load_catalog};

use crate::cli::{SchemaArgs, SchemaDialect};

pub fn run(args: SchemaArgs) -> anyhow::Result<()> {
    let catalog = if args.catalog.exists() {
        load_catalog(&args.catalog)
            .with_context(|| format!("reading catalog {}", args.catalog.display()))?
    } else {
        Catalog::default()
    };

    let dialect = match args.dialect {
        SchemaDialect::Postgres => SqlDialect::Postgres,
        SchemaDialect::Mysql => SqlDialect::Mysql,
        SchemaDialect::Sqlite => SqlDialect::Sqlite,
    };

    let sql = generate_sql_schema(&catalog, dialect);

    match args.output {
        Some(ref path) => {
            if let Some(parent) = path.parent()
                && !parent.as_os_str().is_empty()
                && !parent.exists()
            {
                std::fs::create_dir_all(parent)
                    .with_context(|| format!("creating output directory {}", parent.display()))?;
            }
            std::fs::write(path, &sql)
                .with_context(|| format!("writing {}", path.display()))?;
            println!("infergen: wrote {}", path.display());
        }
        None => print!("{sql}"),
    }

    Ok(())
}
```
