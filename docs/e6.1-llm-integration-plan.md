## Open Questions / Risks

- **JSON extraction from LLM output**: LLMs often wrap JSON in markdown fences. Must strip ```` ```json ... ``` ```` blocks reliably.
- **Rate limits**: Cloud APIs enforce rate limits. Batch size (`default: 10`) keeps single requests small; no retry logic in v1 (soft failure on error).
- **Ollama not running**: `ureq` connect timeout surfaces as `LlmError::Http`. Soft failure — scan completes without LLM pass.
- **Model hallucination**: LLM may suggest invalid snake_case names or fabricate types. Validation: names must match `[a-z][a-z0-9_]*` regex; unknown types are silently dropped.
- **API key in config**: Stored plaintext in `infergen.config.json`. Document that env var alternatives (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`) are preferred.

- Empty response → `LlmError::Empty` raised by callers.

### 5. `crates/infergen-core/src/llm/prompt.rs`

**Purpose:** Build the LLM prompt string from a `RefinementRequest`. Single format works across all providers (all support chat/completion with text prompts).

**Exported symbols:**
```rust
pub fn build_prompt(request: &RefinementRequest) -> String
```

**Key logic:**

The prompt instructs the LLM to:
1. Act as an analytics event naming expert.
2. Review events (name, kind, confidence, source paths, current properties).
3. Suggest a better snake_case name only if meaningfully improved.
4. Add a 5-10 word description for empty descriptions.
5. Infer missing property types (`string` | `number` | `boolean` | `object`).
6. Return **only** a raw JSON object (no markdown, no explanation).

Expected output format embedded in prompt:
```json
{
  "events": [
    {
      "id": "<same id>",
      "name": "<improved_name or null>",
      "description": "<short description or null>",
      "properties": [
        {"name": "<prop>", "prop_type": "<type or null>"}
      ]
    }
  ]
}
```

**Edge cases:**
- Events with `properties: []` → prompt shows empty array, LLM returns empty array.
- `project_context` is derived from detected languages/frameworks, e.g. `"Next.js TypeScript"`.
- Events JSON is serialized with `serde_json::to_string_pretty` for readability.
- PII properties are noted in prompt so LLM treats them as `string` type.

**Dependencies:** `serde_json`, `crate::llm::{RefinementRequest}`

### 6. `crates/infergen-core/src/llm/refine.rs`

**Purpose:** Main entry point for the LLM refinement pass. Filters candidates, batches, calls backend, applies validated refinements back to the catalog.

**Exported symbols:**
```rust
/// Refine low-confidence Proposed events in `catalog` using the LLM.
/// Returns count of events that received at least one change.
/// Soft failure: logs errors per batch and continues.
pub fn refine_catalog(
    catalog: &mut Catalog,
    config: &LlmConfig,
    backend: &dyn LlmBackend,
) -> usize
```

Also a public factory helper for use from scan.rs:
```rust
pub fn refine_catalog_with_config(
    catalog: &mut Catalog,
    config: &LlmConfig,
) -> Result<usize, LlmError>
```
(calls `provider::make_backend(config)?` then `refine_catalog(catalog, config, backend.as_ref())`)

**Key logic:**

```
1. Filter: events where status == Proposed AND confidence < config.confidence_threshold
2. If empty → return 0 (nothing to do)
3. Build project_context string from catalog metadata
4. Chunk filtered event IDs by batch_size
5. For each chunk:
   a. Build Vec<EventInput> from catalog entries
   b. Build RefinementRequest
   c. Call backend.refine_batch(&request)
   d. On Ok: validate and apply response (apply_response)
   e. On Err: eprintln! and continue (never fail scan)
6. Return total count of modified entries
```

**`apply_response` (private fn):**
- For each `EventOutput` in response: find matching `CatalogEntry` by `id`
- Guard: skip if entry `status != Proposed` (never touch Approved/Ignored)
- Name update: apply only if `output.name` is `Some`, non-empty, and matches `^[a-z][a-z0-9_]*$`
- Description update: apply only if `output.description` is `Some`, non-empty, AND `entry.description` is empty (don't overwrite human text)
- Property type update: for each `PropertyOutput`, apply `prop_type` only if the matching property currently has `prop_type: None` (never overwrite known types)

**Edge cases:**
- LLM returns ID not in catalog → silently skip (stale batch).
- LLM returns invalid name (CamelCase, contains spaces, etc.) → skip name change, keep original.
- LLM returns property not in entry → silently skip.
- All batches fail → function returns 0, scan continues.

**Dependencies:** `infergen_types::{Catalog, EventStatus}`, `crate::llm::{LlmBackend, RefinementRequest, EventInput, PropertyInput, LlmError}`, `crate::llm::config::LlmConfig`, `crate::llm::provider`, `crate::llm::prompt::build_prompt`

**Purpose:** Implement `LlmBackend` for Ollama using its native `/api/generate` endpoint.

**Exported symbols:**
```rust
pub struct OllamaBackend {
    base_url: String,    // default "http://localhost:11434"
    model: String,       // default "llama3.2"
    timeout_secs: u64,
}

impl OllamaBackend {
    pub fn new(config: &LlmConfig) -> Self
}

impl LlmBackend for OllamaBackend {
    fn refine_batch(&self, request: &RefinementRequest) -> Result<RefinementResponse, LlmError>
}
```

**Key logic:**

`refine_batch`:
1. Build prompt via `build_prompt(request)`.
2. POST to `{base_url}/api/generate` with body:
   ```json
   {"model": "llama3.2", "prompt": "...", "stream": false}
   ```
3. Parse response body as `{"response": "...", "done": true, ...}`.
4. Extract `response` field → pass to `extract_json` → parse as `RefinementResponse`.

HTTP via `ureq`:
```rust
let body: serde_json::Value = ureq::post(&url)
    .timeout(Duration::from_secs(self.timeout_secs))
    .send_json(ureq::json!({
        "model": self.model,
        "prompt": prompt,
        "stream": false,
    }))
    .map_err(|e| LlmError::Http(e.to_string()))?
    .into_json()
    .map_err(|e| LlmError::Parse(e.to_string()))?;
```

**Edge cases:**
- Ollama not running → `ureq` returns connection refused → `LlmError::Http`.
- `response` field missing from Ollama reply → `LlmError::Parse`.
- Response not valid JSON after extraction → `LlmError::Parse`.

**Dependencies:** `ureq`, `serde_json`, `crate::llm::{LlmBackend, LlmError, RefinementRequest, RefinementResponse, extract_json}`, `crate::llm::prompt::build_prompt`

**Exported symbols:**

```rust
pub struct AnthropicBackend {
    api_key: String,
    model: String,       // default "claude-haiku-4-5-20251001"
    base_url: String,    // default "https://api.anthropic.com"

}

    pub fn new(config: &LlmConfig) -> Result<Self, LlmError>
}

impl LlmBackend for AnthropicBackend {
    fn refine_batch(&self, request: &RefinementRequest) -> Result<RefinementResponse, LlmError>
}
```

**Key logic:**

API key resolution (in order):
1. `config.api_key.clone()` if `Some` and non-empty
2. `std::env::var("ANTHROPIC_API_KEY").ok()` if set and non-empty
3. Return `Err(LlmError::MissingApiKey { provider: "anthropic" })`

`refine_batch`:
1. Build prompt via `build_prompt(request)`.
2. POST to `{base_url}/v1/messages`:
   - Headers: `x-api-key: {key}`, `anthropic-version: 2023-06-01`, `content-type: application/json`
   - Body: `{"model": "...", "max_tokens": 4096, "messages": [{"role": "user", "content": "..."}]}`
3. Parse response: `{"content": [{"type": "text", "text": "..."}], ...}` → extract `content[0].text`.
4. `extract_json(text)` → parse as `RefinementResponse`.

**Edge cases:**
- API returns 401 → `LlmError::Http("401 Unauthorized")`
- API returns 429 (rate limit) → `LlmError::Http("429 Too Many Requests")`
- `content` array empty → `LlmError::Empty`
- `ureq` header injection: `x-api-key` is a custom header, set via `.set("x-api-key", &self.api_key)`

**Default model:** `"claude-haiku-4-5-20251001"` (fast and cheap for naming tasks)

**Dependencies:** `ureq`, `serde_json`, `crate::llm::{LlmBackend, LlmError, RefinementRequest, RefinementResponse, extract_json}`, `crate::llm::prompt::build_prompt`
