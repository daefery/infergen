## Design Decisions Validated

1. **Codegen not Rust HTTP clients** — adapters are TypeScript classes using `fetch()`, emitted by the Rust codegen. Zero new Rust dependencies. ✓
2. **Fire-and-forget** — all adapters call `.catch(() => {})` so a failed tracking call never throws to the caller. ✓
3. **No hardcoded secrets** — all adapters accept secrets at construction time (runtime, not codegen-time). ✓
4. **Determinism** — adapter output is determined solely by provider name; same config = identical output. `cargo test generate_deterministic` still passes. ✓
5. **No adapter section when empty** — `generate_no_providers_no_adapter_section` confirms no section header appears with default config. ✓
6. **`generate --check` unaffected** — existing `--check` tests pass; no-config generate still produces byte-identical output on second run. ✓
