## Open Questions / Risks

- **Runtime stub**: Generated functions throw `"Infergen runtime not configured"`. E3.x replaces this with real provider calls. The stub is intentional and documented in the generated file.
- **`E2.1` vs `E2.2` boundary**: The codegen module delivers both simultaneously. ROADMAP marks both `Done` on completion.
- **Timestamp in header**: NOT included — would cause spurious diffs and break idempotency. E2.3 (Codegen Determinism & Safety) can add `--check` mode.
- **Empty properties interface**: `{}` (empty TypeScript interface). TypeScript accepts any object for `{}`, which is fine at E2.2 scope. E2.2 does not enforce required properties at compile-time for no-property events.
- **Property name collision**: TypeScript identifiers that clash with reserved words are not a concern for v1 — event property names come from source code and pass through the TS identifier unchanged.

## Generated TypeScript Format (specification)

For a catalog with two approved events (`page_viewed` with `route: string`, `user_signed_in` with `method: string` (non-PII) and `email: string` (PII), description "Auth sign-in"):

```typescript
// Auto-generated by Infergen. Do not edit — run `infergen generate` to regenerate.
// infergen-core v0.1.0 · catalog schema v1

/** All tracked event names. */
export type EventName =
  | "page_viewed"
  | "user_signed_in";

// ─── page_viewed ─────────────────────────────────────────────────────────────

/** Properties for the page_viewed event. */
export interface PageViewedProperties {
  /** @type {string} */
  route: string;
}

/**
 * Track a page_viewed event.
 */
export function trackPageViewed(properties: PageViewedProperties): void {
  throw new Error(
    "Infergen runtime not configured. Import and call `configureInfergen()` before tracking events."
  );
}

// ─── user_signed_in ──────────────────────────────────────────────────────────

/** Properties for the user_signed_in event. */
export interface UserSignedInProperties {
  /** @type {string} */
  method: string;
  /**
   * @type {string}
   * @pii Contains personally identifiable information — handle with care.
   */
  email: string;
}

/**
 * Track a user_signed_in event.
 *
 * Auth sign-in
 */
export function trackUserSignedIn(properties: UserSignedInProperties): void {
  throw new Error(
    "Infergen runtime not configured. Import and call `configureInfergen()` before tracking events."
  );
}

/**
 * Track any event by name with fully typed properties.
 *
 * @example
 * track.pageViewed({ route: "/about" });
 * track.userSignedIn({ method: "google", email: "user@example.com" });
 */
export const track = {
  pageViewed: trackPageViewed,
  userSignedIn: trackUserSignedIn,
} as const;
```

**Empty catalog** (no approved events):
```typescript
// Auto-generated by Infergen. Do not edit — run `infergen generate` to regenerate.
// infergen-core v0.1.0 · catalog schema v1

/** All tracked event names. */
export type EventName = never;

/**
 * Track any event by name with fully typed properties.
 * No events are currently approved in the catalog.
 */
export const track = {} as const;
```

#### Private helpers

```rust
/// Convert any identifier to PascalCase: "user_signed_in" → "UserSignedIn".
fn to_pascal_case(name: &str) -> String {
    crate::namer::split_identifier(name)
        .into_iter()
        .map(|w| {
            let mut chars = w.chars();
            match chars.next() {
                None => String::new(),
                Some(f) => f.to_uppercase().to_string() + chars.as_str(),
            }
        })
        .collect()
}

/// Convert any identifier to camelCase: "user_signed_in" → "userSignedIn".
fn to_camel_case(name: &str) -> String {
    let parts = crate::namer::split_identifier(name);
    parts.iter().enumerate().map(|(i, w)| {
        if i == 0 {
            w.clone()
        } else {
            let mut chars = w.chars();
            match chars.next() {
                None => String::new(),
                Some(f) => f.to_uppercase().to_string() + chars.as_str(),
            }
        }
    }).collect()
}

/// Map an EventProperty type string to a TypeScript type string.
fn ts_type(prop_type: Option<&str>) -> &'static str {
    match prop_type {
        Some("string")  => "string",
        Some("number")  => "number",
        Some("boolean") => "boolean",
        _               => "unknown",
    }
}
```

**Note**: `ts_type` returns `"unknown"` for `None` AND any unrecognised string. Pass-through of arbitrary types is not needed at E2.2 scope.

#### `generate_typescript` algorithm

```
1. filter: keep entries where status == Approved, OR (Proposed && config.include_proposed)
2. sort by entry.name ascending (alphabetical, stable)
3. Build output string:
   a. header comment (version line)
   b. EventName union type
   c. for each event: section divider + interface + function
   d. track object
4. return
```

**Header**:
```
// Auto-generated by Infergen. Do not edit — run `infergen generate` to regenerate.
// infergen-core v{VERSION} · catalog schema v{SCHEMA_VERSION}
```
Use `env!("CARGO_PKG_VERSION")` for VERSION, `CATALOG_SCHEMA_VERSION` for schema version.

**EventName type**:
```
// empty:   export type EventName = never;
// N≥1:     export type EventName =\n  | "event_one"\n  | "event_two";
```

**Section divider**: 
```
\n// ─── {name} {bar-to-80-chars}──\n
```
The bar is `─` repeated to fill to 80 chars: `format!("\n// ─── {} {}\n", name, "─".repeat(80 - 7 - name.len().min(73)))`

**Interface**:
```typescript
/** Properties for the {name} event. */
export interface {PascalCase}Properties {
  // for each property:
  //   if pii: /** \n   * @type {ts_type}\n   * @pii ...\n   */
  //   else: /** @type {ts_type} */
  {prop_name}: {ts_type};
}
```
Empty properties → `export interface {PascalCase}Properties {}`

**Function**:
```typescript
/**
 * Track a {name} event.
 *
 * {description if non-empty}
 */
export function track{PascalCase}(properties: {PascalCase}Properties): void {
  throw new Error(
    "Infergen runtime not configured. Import and call `configureInfergen()` before tracking events."
  );
}
```
If description is empty, omit the blank line and description. JSDoc has just `* Track a {name} event.` on one line? No — keep multi-line format always for consistency (E2.2 ergonomics), just without the description paragraph.

Actually for cleanliness: if description is empty, use a single-line doc comment:
```typescript
/** Track a {name} event. */
```
If description is non-empty, use multi-line:
```typescript
/**
 * Track a {name} event.
 *
 * {description}
 */
```

**Track object**:
```typescript
// empty:
/**
 * Track any event by name with fully typed properties.
 * No events are currently approved in the catalog.
 */
export const track = {} as const;

// non-empty:
/**
 * Track any event by name with fully typed properties.
 *
 * @example
 * track.{camel}({ ... });
 */
export const track = {
  {camel}: track{Pascal},
  ...
} as const;
```
`@example` line uses the first event's camelCase name.

### File 5: `crates/infergen-cli/src/commands/generate.rs` (NEW)

**Purpose**: CLI handler for `infergen generate`.

```rust
pub fn run(args: GenerateArgs) -> anyhow::Result<()>
```

**Logic**:
```rust
let catalog = if args.catalog.exists() {
    load_catalog(&args.catalog)
        .with_context(|| format!("reading catalog {}", args.catalog.display()))?
} else {
    Catalog::default()
};

let config = CodegenConfig { include_proposed: args.include_proposed };
let ts = generate_typescript(&catalog, &config);

// count generated events
let generated_count = catalog.events.iter()
    .filter(|e| e.status == EventStatus::Approved || (args.include_proposed && e.status == EventStatus::Proposed))
    .count();

if let Some(parent) = args.output.parent()
    && !parent.as_os_str().is_empty()
    && !parent.exists()
{
    std::fs::create_dir_all(parent)
        .with_context(|| format!("creating output directory {}", parent.display()))?;
}
std::fs::write(&args.output, &ts)
    .with_context(|| format!("writing {}", args.output.display()))?;

println!("infergen: wrote {}  ({} events)", args.output.display(), generated_count);
Ok(())
```
