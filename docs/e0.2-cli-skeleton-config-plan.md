## Resolved decisions

1. **Config lives in `infergen-core`, not the CLI.** The config schema + loader is engine contract (`scan`/`generate`/`check` all consume it). Putting it in core keeps the CLI thin (PRD §8) and lets future non-CLI consumers (editor integration E7.2, web viewer E7.1) reuse it. The CLI only orchestrates.
2. **Config formats: JSON + TOML in E0.2. YAML deferred.** Discovery/loader dispatch on file extension. `init` writes `infergen.config.json` by default (`--format toml` opt-in). Rationale: serde-native, zero-FFI, fully offline. YAML is intentionally **not** added now — the only maintained serde-YAML crate path pulls an *unmaintained* advisory (`serde_yaml`), which would dirty `cargo deny`. The catalog (`.infergen/catalog.yaml`, E1.1) will pick a maintained YAML crate when it actually needs YAML. JS/TS config (`infergen.config.{js,ts}`) is out of scope — it requires a JS runtime, breaking the offline single-binary promise; revisit if demanded.
3. **On-disk config keys are `camelCase`.** Matches the JS/TS-first ecosystem (Next.js slice, `tsconfig.json` familiarity). Applied via `#[serde(rename_all = "camelCase")]`. TOML keys are therefore camelCase too (valid, if slightly less idiomatic) — accepted for one-format-shape consistency.
4. **Config struct field order is TOML-safe.** TOML requires scalars/arrays *before* tables and arrays-of-tables. `Config` fields are ordered: scalars (`schemaVersion`, `catalog`, `output`) → string arrays (`languages`, `frameworks`) → table (`naming`) → array-of-tables (`providers`). This guarantees `toml::to_string_pretty` never mis-nests a later scalar under an earlier table.
5. **`init` does real work; `scan`/`generate`/`check`/`watch` are honest stubs.** `init` detects + writes config (PRD §5.1). The other four parse their args, print `"<command>: not yet implemented — lands in <epic>"`, and **exit 0** (scaffold-friendly, QA-assertable). Exit 0 is safe because no real CI wires `check` yet (it lands in E4.2); the message names the owning epic so behavior is never silently wrong.
6. **No-arg invocation prints a status banner** (version, core engine, schema, hint to run `--help`). Subcommand is `Option<Commands>`. This preserves a friendly bare `infergen` while making the command tree discoverable. The E0.1 default-run test is rewritten accordingly.
7. **Two config-schema versions stay distinct.** `infergen-core::CONFIG_SCHEMA_VERSION` (config file) is separate from `infergen-types::CATALOG_SCHEMA_VERSION` (catalog file). Both start at `1`. Conflating them would couple unrelated format-bump cycles.
8. **Detection is data-driven and tolerant.** Marker files + dependency-manifest inspection, no AST. Malformed/unreadable manifests are skipped (best-effort), never fatal. Only a missing project root is an error. Next.js + TypeScript detection is first-class; Python/Go/Ruby are language-level with a few best-effort framework greps, clearly marked for expansion in M5.

### 3. `crates/infergen-core/src/config.rs` — Score: 97%
**Purpose:** Define the `infergen.config.*` schema and its discover/load/save loader with precise, typed errors.
**Spec — exported surface:**
```rust
//! Infergen project configuration (`infergen.config.{json,toml}`).
//!
//! The config is engine contract: `scan`, `generate`, and `check` all read it.
//! It is discovered in the project root, parsed by file extension, and
//! deserialized with serde defaults so partial configs load cleanly.

use std::path::{Path, PathBuf};
use serde::{Deserialize, Serialize};

/// Candidate config filenames, in discovery precedence order.
pub const CONFIG_FILENAMES: &[&str] = &["infergen.config.json", "infergen.config.toml"];

/// Serialized on-disk format, chosen by `init` / inferred from extension.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ConfigFormat { Json, Toml }

/// Top-level Infergen project configuration.
///
/// Field order is TOML-safe: scalars and arrays precede the `naming` table and
/// the `providers` array-of-tables (see plan decision 4).
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct Config {
    /// Config-file schema version (see `crate::CONFIG_SCHEMA_VERSION`).
    #[serde(default = "default_schema_version")]
    pub schema_version: u32,
    /// Path to the event catalog, relative to project root.
    #[serde(default = "default_catalog_path")]
    pub catalog: PathBuf,
    /// Output directory for the generated SDK, relative to project root.
    #[serde(default = "default_output_dir")]
    pub output: PathBuf,
    /// Detected/declared source languages.
    #[serde(default)]
    pub languages: Vec<crate::detect::Language>,
    /// Detected/declared frameworks.
    #[serde(default)]
    pub frameworks: Vec<crate::detect::Framework>,
    /// Naming-convention settings.
    #[serde(default)]
    pub naming: NamingConfig,
    /// Configured destinations/providers (empty until M3).
    #[serde(default)]
    pub providers: Vec<ProviderConfig>,
}

/// Naming convention for generated event names.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct NamingConfig {
    /// Convention template, e.g. `entity.action.state` (PRD §10 default).
    #[serde(default = "default_convention")]
    pub convention: String,
    /// Casing applied to identifiers, e.g. `snake_case`.
    #[serde(default = "default_case")]
    pub case: String,
}

/// A configured destination/provider entry. Shape firms up in M3 (E3.1).
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ProviderConfig {
    /// Provider identifier, e.g. `posthog`, `segment`, `file`.
    pub name: String,
}
```
**Defaults (free fns + `Default` impls):**
- `default_schema_version() -> u32 { crate::CONFIG_SCHEMA_VERSION }`
- `default_catalog_path() -> PathBuf { ".infergen/catalog.yaml".into() }`
- `default_output_dir() -> PathBuf { "infergen/generated".into() }`
- `default_convention() -> String { "entity.action.state".into() }`
- `default_case() -> String { "snake_case".into() }`
- `impl Default for NamingConfig` → uses the two default fns.
- `impl Default for Config` → all defaults, empty vecs. (Used by `init` as the base before overlaying detection.)

**Methods:**
```rust
impl Config {
    /// Find the first existing config file in `dir` (precedence per CONFIG_FILENAMES).
    pub fn discover(dir: &Path) -> Option<PathBuf>;
    /// Load + parse a config file, dispatching on extension.
    pub fn load(path: &Path) -> Result<Config, Error>;
    /// Discover then load from `dir`; `Err(Error::ConfigNotFound)` if none.
    pub fn load_from_dir(dir: &Path) -> Result<Config, Error>;
    /// Serialize to `path`, format inferred from extension. Creates parent dirs.
    pub fn save(&self, path: &Path) -> Result<(), Error>;
}

impl ConfigFormat {
    /// Infer format from a path's extension. `None` for unknown extensions.
    pub fn from_path(path: &Path) -> Option<ConfigFormat>;
}
```

**Error type — add to `crate::Error` (file 5), referenced here as `Error`:**
```rust
/// I/O failure reading/writing a config file.
Io(#[from] std::io::Error),
/// Config file could not be parsed; carries path + human message.
ConfigParse { path: PathBuf, message: String },
/// No config file found during discovery.
ConfigNotFound { dir: PathBuf },
/// A config already exists at this path (init without --force).
ConfigExists { path: PathBuf },
/// File extension is not a supported config format.
UnsupportedFormat { path: PathBuf },
```

**Key logic / edge cases:**
- `load`: read to string (`?` → `Io`); `match ConfigFormat::from_path`: `Json` → `serde_json::from_str`, `Toml` → `toml::from_str`; map parse errors into `ConfigParse { path, message: e.to_string() }`; `None` → `UnsupportedFormat`.
- `save`: `ConfigFormat::from_path` → `UnsupportedFormat` on miss; serialize (`serde_json::to_string_pretty` with trailing newline, or `toml::to_string_pretty`); create parent dir via `std::fs::create_dir_all` if present; write. Map serialize errors into `ConfigParse` (reuse — message says "serialize").
- `discover`: iterate `CONFIG_FILENAMES`, return first `dir.join(name)` that `.is_file()`.
- camelCase round-trips: `schemaVersion`, `naming`, etc. `#[serde(default)]` on every field makes `{}` (empty JSON) deserialize to all-defaults.
- TOML field ordering (decision 4) verified by the round-trip test.
- `from_path`: lowercase the extension before matching (`JSON` == `json`).
**Dependencies:** `serde`, `serde_json`, `toml`, `crate::detect` (Language/Framework), `crate::Error`.
**Tests (in-file `#[cfg(test)]`, use `tempfile::tempdir`):**
1. `default_config_has_expected_paths` — `Config::default().catalog == PathBuf::from(".infergen/catalog.yaml")`, output, convention, case.
2. `empty_json_deserializes_to_defaults` — `serde_json::from_str::<Config>("{}").unwrap() == Config::default()`.
3. `json_round_trip` — save JSON to tempdir, `load`, assert equal.
4. `toml_round_trip` — save TOML (`infergen.config.toml`), `load`, assert equal (proves field ordering doesn't break serialization).
5. `discover_precedence` — write both `.json` and `.toml`, `discover` returns the `.json` path.
6. `load_from_dir_not_found` — empty dir → `Err(Error::ConfigNotFound { .. })`.
7. `load_unsupported_extension` — `Config::load(Path::new("x.yaml"))` → `Err(Error::UnsupportedFormat { .. })` (no file read needed; checked before/at dispatch — note: check extension first to return UnsupportedFormat without an Io error for a missing path).
8. `load_malformed_json` — write `"{ not json"`, `load` → `Err(Error::ConfigParse { .. })`.
9. `camel_case_keys` — serialize default to JSON string, assert it contains `"schemaVersion"` (not `schema_version`).

> Implementer note for test 7: in `load`, resolve `ConfigFormat::from_path` *before* reading the file so an unsupported extension yields `UnsupportedFormat` rather than an `Io` not-found error.

### 4. `crates/infergen-core/src/detect.rs` — Score: 97%
**Purpose:** Identify project languages + frameworks from marker files and dependency manifests — no AST, fully offline, tolerant of malformed inputs. Powers `init`.
**Spec — exported surface:**
```rust
//! Language & framework auto-detection for `infergen init`.
//!
//! Detection is heuristic and data-driven: it inspects marker files
//! (`package.json`, `tsconfig.json`, `go.mod`, `pyproject.toml`, `Gemfile`, …)
//! and dependency manifests. It never parses source ASTs (that is E0.3) and
//! never fails on a malformed manifest — unreadable inputs are simply skipped.
//! Next.js + TypeScript is first-class (PRD §13); other stacks are best-effort.

use std::path::Path;
use serde::{Deserialize, Serialize};

/// A source language Infergen can target.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case")]
pub enum Language { TypeScript, JavaScript, Python, Go, Ruby }

/// A framework/runtime Infergen has (or will have) an adapter for.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case")]
pub enum Framework {
    NextJs, React, Express, NestJs, Vue, SvelteKit,   // JS/TS
    Django, FastApi, Flask,                            // Python
    Gin, Echo,                                         // Go
    Rails,                                             // Ruby
}

/// Result of scanning a project root for languages + frameworks.
#[derive(Debug, Clone, PartialEq, Eq, Default)]
pub struct DetectionResult {
    /// Detected languages, de-duplicated, in detection order.
    pub languages: Vec<Language>,
    /// Detected frameworks, de-duplicated, in detection order.
    pub frameworks: Vec<Framework>,
}

/// Detect languages + frameworks under `root`.
///
/// # Errors
/// Returns `Error::Io` only if `root` does not exist / is not a directory.
/// Individual unreadable or malformed manifests are skipped, never fatal.
pub fn detect(root: &Path) -> Result<DetectionResult, crate::Error>;
```
**Key logic / edge cases:**
- Guard: if `!root.is_dir()` → `Err(Io(NotFound))`.
- **JS/TS:** if `root/package.json` exists, parse it with `serde_json::Value` (tolerant — on parse error, skip dep inspection but still proceed). Collect keys of `dependencies` + `devDependencies` into a set. Map deps → frameworks:
  - `next` → `NextJs` **and** `React` (Next implies React).
  - `react` (without next) → `React`.
  - `express` → `Express`; `@nestjs/core` → `NestJs`; `vue`/`nuxt` → `Vue`; `@sveltejs/kit`/`svelte` → `SvelteKit`.
  - Language: `TypeScript` if `typescript` is a dep **or** `root/tsconfig.json` exists; else `JavaScript`. (If both `package.json` and `tsconfig.json` but no `typescript` dep, still TypeScript.)
- **Python:** if any of `pyproject.toml`, `requirements.txt`, `setup.py` exist → `Python`. Best-effort framework grep over those files' text (lowercased substring): `django`→`Django`, `fastapi`→`FastApi`, `flask`→`Flask`.
- **Go:** `go.mod` exists → `Go`. Grep text: `gin-gonic/gin`→`Gin`, `labstack/echo`→`Echo`.
- **Ruby:** `Gemfile` exists → `Ruby`. Grep text: `rails`→`Rails`.
- De-dup: push to vec only if not already present (small N, linear `contains` is fine). Deterministic order: languages/frameworks appended in the fixed check order above → reproducible output (PRD determinism spirit).
- Empty/unknown project → `Ok(DetectionResult::default())` (empty vecs). `init` handles this gracefully (writes a minimal config).
- All file reads via `std::fs::read_to_string(...).ok()` so missing/unreadable files yield `None` and are skipped.
**Dependencies:** `serde`, `serde_json`, `crate::Error`.
**Tests (in-file, `tempfile::tempdir`, write fixture files):**
1. `detects_nextjs_typescript` — write `package.json` with `{"dependencies":{"next":"14"},"devDependencies":{"typescript":"5"}}` → languages contains `TypeScript`, frameworks contains `NextJs` and `React`.
2. `detects_javascript_without_typescript` — `package.json` with only `react`, no `typescript`, no tsconfig → `JavaScript` + `React`.
3. `tsconfig_implies_typescript` — `package.json` (react) + empty `tsconfig.json`, no typescript dep → `TypeScript`.
4. `detects_python_fastapi` — `requirements.txt` containing `fastapi==0.110` → `Python` + `FastApi`.
5. `detects_go` — `go.mod` containing `github.com/gin-gonic/gin` → `Go` + `Gin`.
6. `detects_ruby_rails` — `Gemfile` with `gem "rails"` → `Ruby` + `Rails`.
7. `empty_dir_detects_nothing` — empty tempdir → `DetectionResult::default()`.
8. `malformed_package_json_is_tolerant` — `package.json` = `"{ broken"` + `tsconfig.json` present → no panic; still detects `TypeScript` (via tsconfig), frameworks empty.
9. `missing_root_errors` — `detect(Path::new("/no/such/dir"))` → `Err`.
10. `enum_serializes_kebab_case` — `serde_json::to_string(&Language::TypeScript)` == `"\"type-script\""` (locks the on-disk token; consumed by config round-trip).

> Note: `Language::TypeScript` serializes as `"type-script"` under `kebab-case`. Acceptable + stable; test 10 pins it so a future rename is a conscious break.

### 5. `crates/infergen-core/src/lib.rs` — Score: 98%
**Purpose:** Wire the new modules, publish `CONFIG_SCHEMA_VERSION`, and extend `Error` with config variants.
**Spec — changes (additive; keep existing `version()`, `NotImplemented`, re-export, tests):**
```rust
pub mod config;
pub mod detect;

/// Version of the on-disk project config (`infergen.config.*`) schema.
///
/// Distinct from `CATALOG_SCHEMA_VERSION` (the catalog file). Bump on any
/// breaking change to the config format.
```

### 9. `crates/infergen-cli/src/commands/init.rs` — Score: 97%
**Purpose:** The one real command: resolve the target dir, refuse to clobber, detect, build config, write it, print a summary + next step.
**Spec:**
```rust
//! `infergen init` — detect stack and write a project config.

use anyhow::Context;
use infergen_core::config::{Config, ConfigFormat};
use infergen_core::detect;
use crate::cli::{InitArgs, InitFormat};

pub fn run(args: InitArgs) -> anyhow::Result<()> {
    let dir = &args.dir;

    // 1. Refuse to overwrite unless --force.
    if let Some(existing) = Config::discover(dir) {
        if !args.force {
            anyhow::bail!(
                "config already exists at {} (use --force to overwrite)",
                existing.display()
            );
        }
    }

    // 2. Detect stack (tolerant; only errors if dir is missing).
    let result = detect::detect(dir)
        .with_context(|| format!("failed to inspect {}", dir.display()))?;

    // 3. Build config from defaults + detection.
    let mut config = Config::default();
    config.languages = result.languages.clone();
    config.frameworks = result.frameworks.clone();

    // 4. Choose filename + write.
    let (filename, _fmt) = match args.format {
        InitFormat::Json => ("infergen.config.json", ConfigFormat::Json),
        InitFormat::Toml => ("infergen.config.toml", ConfigFormat::Toml),
    };
    let path = dir.join(filename);
    config.save(&path).with_context(|| format!("writing {}", path.display()))?;

    // 5. Summary.
    println!("infergen: wrote {}", path.display());
    print_detection(&result);
    println!("next: run `infergen scan`");
    Ok(())
}
```

## Open questions / risks

1. **`thiserror` + `PathBuf` Display.** `PathBuf` is not `Display`, so every config `Error` variant must format paths via `path.display()` in the `#[error(...)]` positional-arg form. *(Resolved in-plan: file 5 spells out each message.)*
2. **TOML serialization field ordering.** TOML demands scalars/arrays before tables/arrays-of-tables. `Config` field order (decision 4) satisfies this; the `toml_round_trip` test (file 3, test 4) is the guard. *(Resolved in-plan.)*
3. **`deny.toml` license coverage for new deps.** Audited (decision 2 / file 1): serde, serde_json, toml, tempfile + transitives are all MIT/Apache-2.0, already on the allow-list. If `cargo deny check` surfaces an unlisted *permissive* license at build time, add it with a note (never copyleft). *(Resolved in-plan, with fallback.)*
4. **E0.1 default-run test changes.** The old `default_run_prints_scaffold_banner` asserted `"scaffold ready"`, which the E0.2 banner drops. `tests/cli.rs` is rewritten (file 12); the new banner retains `"core engine"`. This is an intentional behavior change for the same binary, not a regression. *(Resolved in-plan.)*
5. **YAML config deferred.** `infergen.config.yaml` is *not* supported in E0.2 (decision 2). If users expect YAML config parity with the catalog, revisit when E1.1 selects a maintained YAML crate. *(Accepted; low risk — JSON/TOML cover the need.)*
6. **`init` leaves a stale config in the other format on `--force` with a changed `--format`.** E.g. existing `.json`, re-init `--force --format toml` writes `.toml` and leaves the `.json`. `discover` precedence would then load `.json`. Minor; acceptable for scaffold. *(Noted; could warn in a later polish.)*
