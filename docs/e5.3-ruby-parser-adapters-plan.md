## Overview

Extends Infergen's scan engine with Ruby support: a text-based line-scanner parser (following the E5.1 Python and E5.2 Go patterns ‚Äî no native Prism FFI needed), a Rails adapter that detects routes, controllers, and Devise auth flows, and a Ruby codegen that emits an idiomatic Ruby module with typed Struct properties, track methods, consent/region runtime, and a `Track` namespace.

`Language::Ruby` and `Framework::Rails` already exist in `detect.rs` with `detect_ruby()` ‚Äî no enum changes required.

## Open Questions / Risks

- **No Prism FFI:** Prism (Ruby's official parser) is a C library ‚Äî cannot be called from Rust without a complex FFI. Following E5.1/E5.2 pattern: line-scanner + regex-free text matching. Confidence scores (0.70‚Äì0.85) reflect text-based limits vs. true AST.
- **`resources` macro:** `resources :users` implicitly generates 5‚Äì7 routes. We emit one event per standard CRUD action (index/show/create/update/destroy) at confidence 0.75.
- **Ruby property name conflicts:** `Struct.new(:method, ...)` shadows `Object#method`. Documented limitation; acceptable for generated code not under user's rubocop.
- **Nested routes:** `namespace :api { resources :users }` ‚Äî nested path prefix not resolved in v1. Path is taken from the inner declaration.
- **Devise:** `devise_for :users` emits `user_signed_in`, `user_signed_out`, `user_registered` events at confidence 0.85.

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum RubyStmt {
    /// `get "/path"` / `get "/path", to: "ctrl#action"` / `get "/path" => "ctrl#action"`
    Route { method: String, path: String, controller_action: String },

}
```

**`RubyParser::scan(source)` implementation:**

Line-by-line scanner. Accumulates `RubyStmt` results. Key patterns:

```
HTTP methods: "get", "post", "put", "patch", "delete", "head", "options", "match"

Route line detection:
  Regex-free: trim ‚Üí check starts_with("get ", "post ", ...) ‚Üí extract quoted path ‚Üí extract action
  
  Pattern 1: `get "/path", to: "ctrl#action"`
    - After method keyword + space: find first quoted string ‚Üí path
    - After path: search for `to: "ctrl#action"` ‚Üí extract quoted value
  
  Pattern 2: `get "/path" => "ctrl#action"`
    - Same path extraction; after `=>` find next quoted string ‚Üí controller_action
  
  Pattern 3: `get "/path"` (no action)
    - path extracted, controller_action = ""
  
  Pattern 4: `get :name` (named route shorthand)
    - path = ":name", controller_action = ""

resources line: `resources :name` or `resources :name, only: [...]`
  - Extract symbol after `resources ` ‚Üí strip leading `:` ‚Üí name
  - (ignore `only:` option ‚Äî we always emit CRUD events)

resource line: `resource :name`
  - Same extraction

devise_for line: `devise_for :users`
  - Extract symbol

ClassDef: line matches `^class\s+(\w+)(\s*<\s*(\w+))?`
  - name = first capture, parent = third capture or ""

MethodDef: line starts with `def ` (after indent strip)
  - Extract method name: chars until `(` or space or end-of-line
  - Skip `initialize`, `self.xxx` (class methods from codegen noise)

Require: `require "..."` or `require '...'`
  - Extract quoted string

ScopeBegin: `namespace :api do` / `scope "/api" do` / `scope :admin do`
  - kind = "namespace" or "scope"
  - label = extracted symbol/path string
```

**`LanguageParser::parse` implementation:**
- Accept only `.rb` extension; return `Err(NotImplemented)` for others.
- Return `ParsedFile { lang: Language::Ruby, source: source.to_owned(), diagnostics: vec![], path: path.to_owned() }`.
- Call `scan()` lazily (not in `parse`; adapters call it via `with_ruby_stmts`).

**Edge cases:**
- Multi-line route declarations (common in real Rails) ‚Üí only single-line detection; complex blocks produce empty results (confidence handles uncertainty).
- `resources :users do ... end` with nested routes ‚Üí detect outer `resources`, ignore inner block.
- Comments (`# get "/path"`) ‚Üí trimmed line starting with `#` ‚Üí skip.
- Heredocs / multiline strings ‚Üí scanner does not track block state; may produce false positives inside string literals (acceptable for heuristic tool).

```rust
/// Scan this file's source as Ruby and call `f` with the structural
/// statements produced by [`ruby::RubyParser::scan`].
///
/// Returns `R::default()` when [`self.lang`][ParsedFile::lang] is not
/// [`Language::Ruby`].
pub fn with_ruby_stmts<F, R>(&self, f: F) -> R
where
    F: FnOnce(&[ruby::RubyStmt]) -> R,
    R: Default,
{
    if self.lang == Language::Ruby {
        let stmts = ruby::RubyParser::scan(&self.source);
        f(&stmts)
    } else {
        R::default()
    }
}
```

Pattern mirrors `with_py_ast` exactly.

### File 3 ‚Äî `crates/infergen-core/src/adapter/rails.rs`

**Path:** `crates/infergen-core/src/adapter/rails.rs`  
**Purpose:** Detect Rails routes, controller actions, and Devise auth flows from Ruby source.


**Detection strategy ‚Äî three passes:**

**Pass 1 ‚Äî routes.rb detection (run when file is a routes file):**

File is a routes file when:
- path ends with `routes.rb`
- OR path contains `config/routes`

For each `RubyStmt`:
- `Route { method, path, controller_action }`:
  - If path starts with `/` or `:` ‚Üí valid route
  - Derive event name via `Namer::derive(NameSignals { route: Some(&path), handler_name: controller_action_action_part, kind: ApiCall, component_name: None })`
    - `controller_action` = `"users#index"` ‚Üí `action_part = "index"` ‚Üí `handler_name = Some("index")`
  - Confidence 0.85 (explicit route)
  - Props: `endpoint: string`, `method: string`
  - `adapter = "rails"`

- `Resources { name }` (plural resource macro):
  - Generate events for CRUD: index, show, create, update, destroy
  - For each action: derive name with route `"/{name}"` (index/create) or `"/{name}/:id"` (show/update/destroy)
  - Confidence 0.75 (implicit from macro)

- `Resource { name }` (singular resource):
  - Generate events for: show, create, update, destroy (no index)
  - Confidence 0.75

- `DeviseFor { resource }`:
  - Emit 3 auth events: `{resource}_signed_in`, `{resource}_signed_out`, `{resource}_registered`
  - EventKind::AuthEvent, confidence 0.85
  - Props: `method: string` (for sign_in/sign_up)

**Pass 2 ‚Äî controller detection (run when file is a controller file):**

File is a controller file when:
- path contains `controllers/` OR path ends with `_controller.rb`

For `ClassDef { name, parent }` where parent contains "Controller" or name ends with "Controller":
- Extract resource name from class name: `UsersController` ‚Üí `users`
- For subsequent `MethodDef { name }` that are CRUD actions or custom actions:
  - CRUD actions (index, show, new, create, edit, update, destroy): confidence 0.70
  - Other methods that don't start with `_` and aren't common helpers: confidence 0.50
  - Derive event: `Namer::derive(NameSignals { route: None, handler_name: Some(&method_name), kind: ApiCall, component_name: Some(&resource_name) })`

**Pass 3 ‚Äî auth detection (run on any Ruby file):**

Check if Devise is imported (`Require { name }` where name contains "devise") ‚Üí on any `devise_for` or Devise module patterns. This is a cross-file signal; for now only detect within the routes file (Pass 1 handles DeviseFor).

**Helper: `controller_resource_name(class_name: &str) -> &str`**
Strips `Controller` suffix and lowercases: `"UsersController"` ‚Üí `"users"`.

**Helper: `is_routes_file(path: &Path) -> bool`**
`path.to_string_lossy().contains("routes") && path.extension().and_then(|e| e.to_str()) == Some("rb")`

**Helper: `is_controller_file(path: &Path) -> bool`**
`path.to_string_lossy().contains("controller") && path.extension() == Some(OsStr::new("rb"))`

**CRUD actions list:**
```rust
const CRUD_ACTIONS: &[&str] = &["index", "show", "new", "create", "edit", "update", "destroy"];
```

**Edge cases:**
- File that is neither routes nor controller ‚Üí empty vec (no false positives from random `.rb` files).
- Empty controller (no methods) ‚Üí no events beyond the class itself.
- `resources` inside `namespace` ‚Üí path prefix missing, events use resource name only (v1 limitation).
- Same route detected in both routes.rb and controller ‚Üí catalog merge handles dedup via stable IDs.
- `match` HTTP method ‚Üí emit as ApiCall with method = "match".

**_dispatch implementation in generated code:**
```ruby
def _dispatch(event_name, props)
  consent, redact, providers = @_mutex.synchronize do
    [@_consent, @_redact_fn, @_providers.dup]
  end
  return if consent == CONSENT_DENIED
  if redact
    props = props.each_with_object({}) { |(k, v), h| h[k] = redact.call(event_name.to_s, k.to_s, v) }
  end
  providers.each { |provider| provider.track(event_name, props) }
end
```

**Per-event track function (with properties):**
```ruby
# Track the user_signed_in event.

end
```

**Per-event track function (no properties):**
```ruby
# Track the click_happened event.
```
